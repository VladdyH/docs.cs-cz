## <a name="security-model"></a><span data-ttu-id="f5d90-130">Model zabezpečení</span><span class="sxs-lookup"><span data-stu-id="f5d90-130">Security Model</span></span>  
 <span data-ttu-id="f5d90-131">Rovnocenného kanálu poskytuje možnost zabezpečení jednotlivých propojení mezi rovnocennými počítači.</span><span class="sxs-lookup"><span data-stu-id="f5d90-131">Peer Channel provides the ability to secure the individual links between peers.</span></span> <span data-ttu-id="f5d90-132">To znamená, že zprávu nikdy toků v rámci nezabezpečené připojení (z pohledu aplikace).</span><span class="sxs-lookup"><span data-stu-id="f5d90-132">This means that a message never flows on an unsecured link (from the application perspective).</span></span> <span data-ttu-id="f5d90-133">Každé propojení (kanál mezi dvěma partnery transport) je interně zabezpečené, pomocí zabezpečení TLS (Transport Layer).</span><span class="sxs-lookup"><span data-stu-id="f5d90-133">Internally, each link (a transport channel between two peers) is secured using Transport Layer Security (TLS).</span></span> <span data-ttu-id="f5d90-134">To znamená, že když odesílatele vytvoří a odešle zprávu, se odešle přes zabezpečené přenosu na každý z jeho okamžitou partnerské uzly, kteří přístup ke zprávě a pak odeslat zprávu svým spolupracovníkům okamžitou přes zabezpečené připojení.</span><span class="sxs-lookup"><span data-stu-id="f5d90-134">This means that when a sender composes and sends a message, it is sent over secure transport to each of its immediate peers, who access the message, and in turn send the message to their immediate peers over secure connections.</span></span> <span data-ttu-id="f5d90-135">Tato zabezpečení funguje pouze na přenos úrovni a je nezávislý na modely zabezpečení zpráv.</span><span class="sxs-lookup"><span data-stu-id="f5d90-135">This security only works at the transport level and is independent of the message security models.</span></span>  
  
 <span data-ttu-id="f5d90-136">Rovnocenného kanálu také poskytuje způsob, jak zabezpečit zprávy nezávisle na zabezpečení přenosu, který používá.</span><span class="sxs-lookup"><span data-stu-id="f5d90-136">Peer Channel also provides a way to secure messages independently of the transport security used.</span></span> <span data-ttu-id="f5d90-137">V tomto modelu je zabezpečená zpráva na zdroji pomocí tokenu zabezpečení zdroje, ačkoliv je aktuálně jedinou certifikáty X.509.</span><span class="sxs-lookup"><span data-stu-id="f5d90-137">In this model, the message is secured at the source using the source’s security token, although currently only X.509 certificates are supported.</span></span> <span data-ttu-id="f5d90-138">Zabezpečené zprávy je pak přenést přes síť partnera.</span><span class="sxs-lookup"><span data-stu-id="f5d90-138">The secured message is then transmitted over the peer network.</span></span> <span data-ttu-id="f5d90-139">Každý přijímající partnera můžete ověřit pravost zdroje.</span><span class="sxs-lookup"><span data-stu-id="f5d90-139">Each receiving peer can verify the authenticity of the source.</span></span> <span data-ttu-id="f5d90-140">Všimněte si, že je zpráva zabezpečené tak, aby prostředníci nelze manipulovat s ním.</span><span class="sxs-lookup"><span data-stu-id="f5d90-140">Note that the message is secured so that intermediaries cannot tamper with it.</span></span>  
  
 <span data-ttu-id="f5d90-141">K dosažení důvěrnost, můžete použít aplikace zabezpečení přenosu s silné skupiny členství schémata znemožníte neoprávněný přístup ke zprávě.</span><span class="sxs-lookup"><span data-stu-id="f5d90-141">To achieve confidentiality, applications can employ transport security with strong group membership schemes to prevent unauthorized access to the message.</span></span>  
  
 <span data-ttu-id="f5d90-142">Rovnocenného kanálu nevyžaduje konkrétní identity modelu, tak dlouho, dokud aplikace zvolí jeden z podporované typy tokenů.</span><span class="sxs-lookup"><span data-stu-id="f5d90-142">Peer Channel does not require a specific identity model as long as the application chooses one of the supported token types.</span></span> <span data-ttu-id="f5d90-143">Aplikace úplně vlastní životní cyklus těchto identit a ověření rozhodnutí.</span><span class="sxs-lookup"><span data-stu-id="f5d90-143">Applications completely own the life cycle of these identities and authentication decisions.</span></span>
