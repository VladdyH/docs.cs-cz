### <a name="transfer-security"></a><span data-ttu-id="50682-172">Zabezpečení přenosu</span><span class="sxs-lookup"><span data-stu-id="50682-172">Transfer Security</span></span>  
 <span data-ttu-id="50682-173">Zabezpečení přenosu zahrnuje tři hlavní zabezpečení funkce: integrity, šifrování a ověřování.</span><span class="sxs-lookup"><span data-stu-id="50682-173">Transfer security encompasses three major security functions: integrity, confidentiality, and authentication.</span></span> <span data-ttu-id="50682-174">*Integrita* je schopnost rozpoznat, zda zpráva bylo manipulováno.</span><span class="sxs-lookup"><span data-stu-id="50682-174">*Integrity* is the ability to detect whether a message has been tampered with.</span></span> <span data-ttu-id="50682-175">*Důvěrnost* je schopnost zachovat zprávu nejde přečíst nikdo jiný než zamýšlený příjemce; toho je dosaženo pomocí šifrování.</span><span class="sxs-lookup"><span data-stu-id="50682-175">*Confidentiality* is the ability to keep a message unreadable by anyone other than the intended recipient; this is achieved through cryptography.</span></span> <span data-ttu-id="50682-176">*Ověřování* je schopnost ověření uváděné identity.</span><span class="sxs-lookup"><span data-stu-id="50682-176">*Authentication* is the ability to verify a claimed identity.</span></span> <span data-ttu-id="50682-177">Tyto tři funkce společně pomáhají zajistit, aby bezpečně doručování zpráv z jednoho bodu do jiného.</span><span class="sxs-lookup"><span data-stu-id="50682-177">Together, these three functions help to ensure that messages securely arrive from one point to another.</span></span>  
  
#### <a name="transport-and-message-security-modes"></a><span data-ttu-id="50682-178">Přenos a režimy zabezpečení zpráv</span><span class="sxs-lookup"><span data-stu-id="50682-178">Transport and Message Security Modes</span></span>  
 <span data-ttu-id="50682-179">Dva hlavní mechanismy slouží k implementaci zabezpečení přenosu ve WCF: *přenosu* režimu zabezpečení a *zpráva* režim zabezpečení.</span><span class="sxs-lookup"><span data-stu-id="50682-179">Two main mechanisms are used to implement transfer security in WCF: *transport* security mode and *message* security mode.</span></span>  
  
-   <span data-ttu-id="50682-180">*Režim zabezpečení přenosu* používá protokol transportní vrstvy, jako je například HTTPS, abyste dosáhli zabezpečení přenosu.</span><span class="sxs-lookup"><span data-stu-id="50682-180">*Transport security mode* uses a transport-level protocol, such as HTTPS, to achieve transfer security.</span></span> <span data-ttu-id="50682-181">Režim přenosu má výhodu v podobě se široce přijímaná, k dispozici na mnoha platformách a méně výpočetně komplexní.</span><span class="sxs-lookup"><span data-stu-id="50682-181">Transport mode has the advantage of being widely adopted, available on many platforms, and less computationally complex.</span></span> <span data-ttu-id="50682-182">Má však nevýhodou zabezpečení zpráv jenom z typu point-to-point.</span><span class="sxs-lookup"><span data-stu-id="50682-182">However, it has the disadvantage of securing messages only from point-to-point.</span></span>  
  
-   <span data-ttu-id="50682-183">*Režim zabezpečení zprávy*na druhé straně, používá zabezpečení WS (a dalších specifikacích) k implementaci zabezpečení přenosu.</span><span class="sxs-lookup"><span data-stu-id="50682-183">*Message security mode*, on the other hand, uses WS-Security (and other specifications) to implement transfer security.</span></span> <span data-ttu-id="50682-184">Vzhledem k tomu, že zabezpečení zpráv se použije přímo ke zprávám SOAP a se nachází v obálky protokolu SOAP, společně s dat aplikací, má výhodu v podobě probíhá přenos nezávislý, více rozšiřitelný a zajistit – koncové zabezpečení (oproti point-to-point); má nevýhodou je několikrát pomalejší než režim zabezpečení přenosu, protože se musí řešit XML povaha protokolu SOAP zprávy.</span><span class="sxs-lookup"><span data-stu-id="50682-184">Because the message security is applied directly to the SOAP messages and is contained inside the SOAP envelopes, together with the application data, it has the advantage of being transport protocol-independent, more extensible, and ensuring end-to-end security (versus point-to-point); it has the disadvantage of being several times slower than transport security mode because it has to deal with the XML nature of the SOAP messages.</span></span>  
  
 <span data-ttu-id="50682-185">Další informace o těchto rozdílech najdete v tématu [zabezpečení služeb a klientů](../../../../docs/framework/wcf/feature-details/securing-services-and-clients.md).</span><span class="sxs-lookup"><span data-stu-id="50682-185">For more information about these differences, see [Securing Services and Clients](../../../../docs/framework/wcf/feature-details/securing-services-and-clients.md).</span></span>  
  
 <span data-ttu-id="50682-186">Třetí režim zabezpečení používá oba režimy předchozí a přináší výhody obou.</span><span class="sxs-lookup"><span data-stu-id="50682-186">A third security mode uses both previous modes and brings advantages of both.</span></span> <span data-ttu-id="50682-187">Tento režim se nazývá `TransportWithMessageCredential`.</span><span class="sxs-lookup"><span data-stu-id="50682-187">This mode is called `TransportWithMessageCredential`.</span></span> <span data-ttu-id="50682-188">V tomto režimu zabezpečení zpráv se používá k ověření klienta a zabezpečení přenosu se používá k ověření serveru a zajištění důvěrnosti zpráv a integrita.</span><span class="sxs-lookup"><span data-stu-id="50682-188">In this mode, message security is used to authenticate the client and transport security is used to authenticate the server and provide message confidentiality and integrity.</span></span> <span data-ttu-id="50682-189">Díky tomu `TransportWithMessageCredential` režim zabezpečení je téměř tak rychlý jako režim zabezpečení přenosu a zajišťuje rozšiřitelnost ověřování klienta stejným způsobem jako zabezpečení zpráv.</span><span class="sxs-lookup"><span data-stu-id="50682-189">Thanks to this, the `TransportWithMessageCredential` security mode is almost as fast as transport security mode and provides client authentication extensibility in the same way as message security.</span></span> <span data-ttu-id="50682-190">Ale na rozdíl od režim zabezpečení zprávy, neposkytuje dokončení – koncové zabezpečení.</span><span class="sxs-lookup"><span data-stu-id="50682-190">However, unlike message security mode, it does not provide complete end-to-end security.</span></span>  
  
### <a name="access-control"></a><span data-ttu-id="50682-191">Access Control</span><span class="sxs-lookup"><span data-stu-id="50682-191">Access Control</span></span>  
 <span data-ttu-id="50682-192">*Řízení přístupu* je také označován jako autorizace.</span><span class="sxs-lookup"><span data-stu-id="50682-192">*Access control* is also known as authorization.</span></span> <span data-ttu-id="50682-193">*Autorizace* umožňuje různým uživatelům různých oprávnění k zobrazení dat.</span><span class="sxs-lookup"><span data-stu-id="50682-193">*Authorization* allows different users to have different privileges to view data.</span></span> <span data-ttu-id="50682-194">Například protože společnosti lidských zdrojů soubory obsahují zaměstnanec citlivá data, jsou povolena pouze správci zobrazíte údaje o zaměstnancích.</span><span class="sxs-lookup"><span data-stu-id="50682-194">For example, because a company's human resources files contain sensitive employee data, only managers are allowed to view employee data.</span></span> <span data-ttu-id="50682-195">Správci navíc můžete zobrazit jenom data pro jejich přímé podřízené.</span><span class="sxs-lookup"><span data-stu-id="50682-195">Further, managers can view only data for their direct reports.</span></span> <span data-ttu-id="50682-196">V takovém případě řízení přístupu podle rolí ("správce") jak konkrétní identity manager (Chcete-li zabránit prohlížení záznamy o jiný správce zaměstnanci jeden správce).</span><span class="sxs-lookup"><span data-stu-id="50682-196">In this case, access control is based on both the role ("manager") as well as the specific identity of the manager (to prevent one manager from looking at another manager's employee records).</span></span>  
  
 <span data-ttu-id="50682-197">Ve službě WCF, funkce řízení přístupu je zajišťována prostřednictvím integrace s common language runtime (CLR) <xref:System.Security.Permissions.PrincipalPermissionAttribute> přes sadu rozhraní API, které jsou známé jako *modelu identity*.</span><span class="sxs-lookup"><span data-stu-id="50682-197">In WCF, access control features are provided through integration with the common language runtime (CLR) <xref:System.Security.Permissions.PrincipalPermissionAttribute> and through a set of APIs known as the *identity model*.</span></span> <span data-ttu-id="50682-198">Podrobnosti o řízení přístupu a autorizace na základě deklarací identity najdete v tématu [rozšíření zabezpečení](../../../../docs/framework/wcf/extending/extending-security.md).</span><span class="sxs-lookup"><span data-stu-id="50682-198">For details about access control and claims-based authorization, see [Extending Security](../../../../docs/framework/wcf/extending/extending-security.md).</span></span>
