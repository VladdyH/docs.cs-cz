<span data-ttu-id="4473f-103">Za normálních okolností při hostování služby Windows Communication Foundation (WCF) v rámci Internetové informační služby (IIS) nebo služby Aktivace procesů systému Windows (WAS), je nutné zadat soubor .svc.</span><span class="sxs-lookup"><span data-stu-id="4473f-103">Normally when hosting a Windows Communication Foundation (WCF) service under Internet Information Services (IIS) or Windows Process Activation Service (WAS), you must provide a .svc file.</span></span> <span data-ttu-id="4473f-104">Soubor .svc obsahuje název služby a objekt pro vytváření hostitele volitelné vlastní službu.</span><span class="sxs-lookup"><span data-stu-id="4473f-104">The .svc file contains the name of the service and an optional custom service host factory.</span></span> <span data-ttu-id="4473f-105">Tento další soubor přidá režijní náklady na možnosti správy.</span><span class="sxs-lookup"><span data-stu-id="4473f-105">This additional file adds manageability overhead.</span></span> <span data-ttu-id="4473f-106">Aktivace podle konfigurace funkce eliminuje požadavek na soubor .svc a proto přidružené režijní náklady.</span><span class="sxs-lookup"><span data-stu-id="4473f-106">The configuration-based activation feature removes the requirement to have a .svc file and therefore the associated overhead.</span></span>  
  
## <a name="configuration-based-activation"></a><span data-ttu-id="4473f-107">Aktivace podle konfigurace</span><span class="sxs-lookup"><span data-stu-id="4473f-107">Configuration-Based Activation</span></span>  
 <span data-ttu-id="4473f-108">Aktivace podle konfigurace trvá metadata, která umožňuje umístit do souboru .svc a umístí jej do souboru Web.config.</span><span class="sxs-lookup"><span data-stu-id="4473f-108">Configuration-based activation takes the metadata that used to be placed in the .svc file and places it in the Web.config file.</span></span> <span data-ttu-id="4473f-109">V rámci <`serviceHostingEnvironment`> elementu je <`serviceActivations`> elementu.</span><span class="sxs-lookup"><span data-stu-id="4473f-109">Within the<`serviceHostingEnvironment`> element there is a <`serviceActivations`> element.</span></span> <span data-ttu-id="4473f-110">V rámci <`serviceActivations`> element jsou jeden nebo více <`add`> elementy, jeden pro každou hostovanou službu.</span><span class="sxs-lookup"><span data-stu-id="4473f-110">Within the <`serviceActivations`> element are one or more <`add`> elements, one for each hosted service.</span></span> <span data-ttu-id="4473f-111"><`add`> Element obsahuje atributy, které umožňují nastavit relativní adresu pro službu a typ služby nebo vytváření hostitele služby.</span><span class="sxs-lookup"><span data-stu-id="4473f-111">The <`add`> element contains attributes that let you set the relative address for the service and the service type or a service host factory.</span></span> <span data-ttu-id="4473f-112">Následující příklad kódu konfigurace ukazuje, jak se používá v této části.</span><span class="sxs-lookup"><span data-stu-id="4473f-112">The following configuration example code shows how this section is used.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="4473f-113">Každý <`add`> element musí specifikovat, služby nebo atributu objektu pro vytváření.</span><span class="sxs-lookup"><span data-stu-id="4473f-113">Each <`add`> element must specify a service or a factory attribute.</span></span> <span data-ttu-id="4473f-114">Zadání služby a objektu pro vytváření atributy je povoleno.</span><span class="sxs-lookup"><span data-stu-id="4473f-114">Specifying both service and factory attributes is allowed.</span></span>  
  
```xml  
<serviceHostingEnvironment>  
  <serviceActivations>  
    <add relativeAddress="MyServiceAddress" service="Service" factory="MyServiceHostFactory"/>  
  </serviceActivations>  
</serviceHostingEnvironment>  
```  
  
 <span data-ttu-id="4473f-115">Pomocí této v souboru Web.config můžete umístit zdrojový kód služby v adresáři App_Code dodržela sestavení v adresáři Bin aplikace nebo aplikace.</span><span class="sxs-lookup"><span data-stu-id="4473f-115">With this in the Web.config file, you can place the service source code in the App_Code directory of the application or a complied assembly in the Bin directory of the application.</span></span>
