<span data-ttu-id="0cad5-103">Můžete používat Windows Communication Foundation (WCF) pro nový vývoj nadále zachovat stávající aplikace vyvinuté pomocí technologie ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="0cad5-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="0cad5-104">Protože WCF má být nejvhodnější volbou pro usnadnění komunikace s aplikace vytvořené pomocí rozhraní .NET Framework v žádném scénáři, mohl sloužit jako standardní nástroj pro řešení širokou škálu problémy s komunikací softwaru tak, že technologie ASP.NET nelze provést.</span><span class="sxs-lookup"><span data-stu-id="0cad5-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>  
  
 <span data-ttu-id="0cad5-105">Nové aplikace WCF můžete nasadit na stejný počítače jako existující webové služby ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="0cad5-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="0cad5-106">Pokud existujících webových služeb ASP.NET použijte verzi rozhraní .NET Framework verze 2.0, můžete použít ASP.NET IIS Registration Tool selektivně nasazení do aplikace služby IIS, ve kterých jsou nové aplikace WCF pro hostování rozhraní .NET Framework 2.0.</span><span class="sxs-lookup"><span data-stu-id="0cad5-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="0cad5-107">Tento nástroj je popsána v [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](http://go.microsoft.com/fwlink/?LinkId=94687), a je součástí uživatelské rozhraní konzoly pro správu služby IIS 6.0.</span><span class="sxs-lookup"><span data-stu-id="0cad5-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](http://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>  
  
 <span data-ttu-id="0cad5-108">WCF slouží k přidání nových funkcí do existující webových služeb ASP.NET přidáním služby WCF, které jsou nakonfigurovány na spuštění v režimu kompatibility ASP.NET do stávajících aplikací ASP.NET – webové služby ve službě IIS.</span><span class="sxs-lookup"><span data-stu-id="0cad5-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="0cad5-109">Z důvodu režimu kompatibility ASP.NET, můžete používat a aktualizovat stejné informace o stavu aplikace jako existující kódu ASP.NET pomocí kód pro nové služby WCF <xref:System.Web.HttpContext> třídy.</span><span class="sxs-lookup"><span data-stu-id="0cad5-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="0cad5-110">Aplikace můžou taky sdílet stejné knihovny tříd.</span><span class="sxs-lookup"><span data-stu-id="0cad5-110">The applications can also share the same class libraries.</span></span>  
  
 <span data-ttu-id="0cad5-111">Klienti WCF pomocí webových služeb ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="0cad5-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="0cad5-112">Služby WCF, které jsou nakonfigurovány s <xref:System.ServiceModel.BasicHttpBinding> mohou být využívána klienty webové služby ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="0cad5-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="0cad5-113">Webových služeb ASP.NET může existovat společně s aplikací služby WCF a WCF lze použít i pro přidání funkcí do existující webové služby ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="0cad5-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="0cad5-114">Zadány všechny tyto způsoby, ve kterých WCF a ASP.NET Web services můžete použít společně, můžete migrovat webových služeb ASP.NET na WCF pouze v případě, že budete potřebovat funkce, které jsou k dispozici službami WCF a není ASP.NET Web.</span><span class="sxs-lookup"><span data-stu-id="0cad5-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>  
  
 <span data-ttu-id="0cad5-115">I v několika případech, kdy je nezbytné pečlivě zvažte, že migrace kód z jednoho technologie do jiného je málokdy správný přístup.</span><span class="sxs-lookup"><span data-stu-id="0cad5-115">Even in the few cases where it is necessary, carefully consider that migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="0cad5-116">Důvodem pro přijetí nové technologie je splňují nové požadavky, které nelze splnit s dřívější technologie a v takovém případě správný krokem je návrhu nové řešení ke splnění nově rozšířit sadu požadavků.</span><span class="sxs-lookup"><span data-stu-id="0cad5-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="0cad5-117">Nové výhody návrhu z vašich zkušeností s stávajícího systému a moudrý získávají vzhledem k tomu, že byla navržená tak, že systému.</span><span class="sxs-lookup"><span data-stu-id="0cad5-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="0cad5-118">Nový design použít také veškeré funkce nové technologie a ne reprodukci staré návrhu na nové platformě.</span><span class="sxs-lookup"><span data-stu-id="0cad5-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="0cad5-119">Po vytváření prototypů klíčové prvky nového návrhu bude jednodušší opětovné použití kódu z existujícího systému v rámci nového.</span><span class="sxs-lookup"><span data-stu-id="0cad5-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>  
  
 <span data-ttu-id="0cad5-120">Pro několik případů, kde portování z webových služeb ASP.NET na WCF je správným řešením, v následujícím oddílu najdete některé pokyny o tom, jak pokračovat.</span><span class="sxs-lookup"><span data-stu-id="0cad5-120">For the few cases where porting from ASP.NET Web services to WCF is the correct solution, the following section provides some guidance on how to proceed.</span></span> <span data-ttu-id="0cad5-121">Není Rady, jak migrovat služby a jak migraci klientů.</span><span class="sxs-lookup"><span data-stu-id="0cad5-121">There is advice on how to migrate services, and how to migrate clients.</span></span>  
  
 <span data-ttu-id="0cad5-122">Dokončení analýzy o tom, jak migrovat existující webové služby ASP.NET na WCF najdete v tématu [webových služeb ASP.NET a Windows Communication Foundation](http://go.microsoft.com/fwlink/?LinkID=71761).</span><span class="sxs-lookup"><span data-stu-id="0cad5-122">For a complete analysis on how to migrate existing ASP.NET Web Services to WCF please see [ASP.NET Web Services and the Windows Communication Foundation](http://go.microsoft.com/fwlink/?LinkID=71761).</span></span> <span data-ttu-id="0cad5-123">Tato část popisuje, jak implementovat kompatibilní služby WCF z metadat pro vás webovou službu ASP.NET a jak migrovat ASP.NET Web kód klienta a služby WCF.</span><span class="sxs-lookup"><span data-stu-id="0cad5-123">This section describes how to implement a compliant WCF service from the metadata for you ASP.NET Web Service, and how to migrate ASP.NET Web service and client code to WCF.</span></span>
