# <a name="wif-code-sample-index"></a>Index ukázkový kód WIF
Následují ukázky kódu pro Windows Identity Foundation 4.5:  
  
-   [ClaimsAwareWebApp](http://go.microsoft.com/fwlink/?LinkID=248405) – tento příklad znázorňuje základní použití externalization ověřování (pro místní test služby tokenů zabezpečení z Identity a přístup pro sadu Visual Studio 11) na klasické aplikace ASP.NET (na rozdíl od webový server).  
  
-   [ClaimsAwareWebService](http://go.microsoft.com/fwlink/?LinkID=248406) – tento příklad znázorňuje základní použití ověřování externalization na klasické služby WCF.  
  
-   [ClaimsAwareMvcApplication](http://go.microsoft.com/fwlink/?LinkID=248407) – Tato ukázka ukazuje, jak integrovat WIF MVC, včetně jiných hromadný ochrany a kód, který splňuje ověřování založené na formulářích přesměruje mimo řadičem přihlášení.  
  
-   [ClaimsAwareWebFarm](http://go.microsoft.com/fwlink/?LinkID=248408) – tento příklad znázorňuje mezipaměti připravené relace farmy (na rozdíl od tokenreplycache) tak, aby můžete odkazem místo výměna velkých souborů cookie relací. Také ukazuje jednodušší způsob zabezpečení soubory cookie ve farmě.  
  
-   [ClaimsAwareFormsAuthentication](http://go.microsoft.com/fwlink/?LinkID=248409) -Tento velmi jednoduchý příklad znázorňuje, že v rozhraní .NET 4.5 dostanete deklarace identity ve vašem objekty bez ohledu na to, jak ověřovat uživatele.  
  
-   [ClaimsBasedAuthorization](http://go.microsoft.com/fwlink/?LinkID=248410)– této ukázky ukazuje, jak používat třídu CLaimsAuthorizationManager a ClaimsAuthorizationModule na uplatnění zásad autorizace.  
  
-   [FederationMetadata](http://go.microsoft.com/fwlink/?LinkID=248411) – tento příklad znázorňuje dynamické generování (na vlastní STS) a dynamické používání (na aplikace předávající strany) dokumenty metadat.  
  
-   [CustomToken](http://go.microsoft.com/fwlink/?LinkID=248412) – tento příklad ukazuje, jak vytvořit vlastní typ tokenu jednoduchých webových tokenů (SWT).
