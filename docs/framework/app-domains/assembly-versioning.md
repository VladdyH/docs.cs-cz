## <a name="version-information"></a><span data-ttu-id="3d188-117">Informace o verzi</span><span class="sxs-lookup"><span data-stu-id="3d188-117">Version Information</span></span>

### <a name="assembly-version-number"></a><span data-ttu-id="3d188-122">Číslo verze sestavení</span><span class="sxs-lookup"><span data-stu-id="3d188-122">Assembly Version Number</span></span>  
 <span data-ttu-id="3d188-123">Každé sestavení má číslo verze jako součást svou identitu.</span><span class="sxs-lookup"><span data-stu-id="3d188-123">Each assembly has a version number as part of its identity.</span></span> <span data-ttu-id="3d188-124">Jako takový dvou sestavení, které se liší podle čísla verze jsou považovány za modulem runtime úplně jiné sestavení.</span><span class="sxs-lookup"><span data-stu-id="3d188-124">As such, two assemblies that differ by version number are considered by the runtime to be completely different assemblies.</span></span> <span data-ttu-id="3d188-125">Toto číslo verze je fyzicky reprezentována jako řetězec sestávající ze čtyř částí v následujícím formátu:</span><span class="sxs-lookup"><span data-stu-id="3d188-125">This version number is physically represented as a four-part string with the following format:</span></span>  
  
 <span data-ttu-id="3d188-126">\<*hlavní verze*>.\< *podverze*>.\< *číslo sestavení*>.\< *revize*></span><span class="sxs-lookup"><span data-stu-id="3d188-126">\<*major version*>.\<*minor version*>.\<*build number*>.\<*revision*></span></span>  
  
 <span data-ttu-id="3d188-127">Například verze 1.5.1254.0 označuje 1 jako hlavní verze 5 jako podverzi, 1254 jako číslo sestavení a 0 jako číslo revize.</span><span class="sxs-lookup"><span data-stu-id="3d188-127">For example, version 1.5.1254.0 indicates 1 as the major version, 5 as the minor version, 1254 as the build number, and 0 as the revision number.</span></span>  
  
 <span data-ttu-id="3d188-128">Číslo verze je uložené v manifestu sestavení společně s dalšími informacemi o identitě včetně na název sestavení a veřejný klíč, a také informace o relacích a identitách jiných sestavení spojených s aplikací.</span><span class="sxs-lookup"><span data-stu-id="3d188-128">The version number is stored in the assembly manifest along with other identity information, including the assembly name and public key, as well as information on relationships and identities of other assemblies connected with the application.</span></span>  
  
 <span data-ttu-id="3d188-129">Pokud sestavení vytvořené, vývoj nástroj zaznamenává informace o závislostech pro každé sestavení, který se odkazuje v manifestu sestavení.</span><span class="sxs-lookup"><span data-stu-id="3d188-129">When an assembly is built, the development tool records dependency information for each assembly that is referenced in the assembly manifest.</span></span> <span data-ttu-id="3d188-130">Modul runtime používá tato čísla verze ve spojení s informace o konfiguraci, která nastavuje správce, aplikace nebo vydavatel, k načtení správné verze odkazovaného sestavení.</span><span class="sxs-lookup"><span data-stu-id="3d188-130">The runtime uses these version numbers, in conjunction with configuration information set by an administrator, an application, or a publisher, to load the proper version of a referenced assembly.</span></span>
