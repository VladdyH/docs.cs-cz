# <a name="how-to-make-thread-safe-calls-to-windows-forms-controls"></a>Postupy: Volání (s bezpečným vlákny) ovládacích prvků Windows Forms
Pokud používáte více vláken zlepšit výkon aplikací Windows Forms, ujistěte se, abyste vytvořili volání pro vaše ovládací prvky způsobem bezpečné pro přístup z více vláken.  
  
 Přístup k ovládacím prvkům Windows Forms ze své podstaty není bezpečné pro přístup z více vláken. Pokud máte dva nebo více podprocesů manipulace s stavu ovládacího prvku, je možné vynutit ovládacího prvku v nekonzistentním stavu. Je možné, například časování a blokování jiných chyby související s přístup z více vláken. Je důležité se ujistit, že se provádí přístup k ovládacím prvkům způsobem bezpečné pro přístup z více vláken.  
  
 Není bezpečné volat z vlákna, než ten, který vytvořil ovládacího prvku bez použití ovládacího prvku <xref:System.Windows.Forms.Control.Invoke%2A> metoda. Následuje příklad volání, která není bezpečná pro přístup z více vláken.  
  
```csharp  
// This event handler creates a thread that calls a   
// Windows Forms control in an unsafe way.  
private void setTextUnsafeBtn_Click(  
    object sender,   
    EventArgs e)  
{  
    this.demoThread =   
        new Thread(new ThreadStart(this.ThreadProcUnsafe));  
  
    this.demoThread.Start();  
}  
  
// This method is executed on the worker thread and makes  
// an unsafe call on the TextBox control.  
private void ThreadProcUnsafe()  
{  
    this.textBox1.Text = "This text was set unsafely.";  
}  
```

Když aplikace běží v ladicím programu a vlákna než ten, který vytvoří ovládacího prvku se pokusí zavolat, že ovládací prvek, ladicího programu vyvolá <xref:System.InvalidOperationException> se zobrazí zpráva "ovládací prvek *název ovládacího prvku* k němu přistupovat z vlákna než, který byl vytvořen na vlákno."  
  
 Tato došlo k výjimce spolehlivě během ladění a za určitých okolností za běhu. Můžete se setkat výjimku při ladění aplikace, které jste napsali s [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] před verzí [!INCLUDE[dnprdnext](../../../../includes/dnprdnext-md.md)]. Důrazně doporučujeme o řešení tohoto problému, když se zobrazovat, ale můžete ji zakázat nastavením <xref:System.Windows.Forms.Control.CheckForIllegalCrossThreadCalls%2A> vlastnost `false`. To způsobí, že vaše řízení spustit jako bude spuštěná v sadě Visual Studio .NET 2003 a [!INCLUDE[net_v11_short](../../../../includes/net-v11-short-md.md)].

## <a name="making-thread-safe-calls-to-windows-forms-controls"></a>Volání bezpečného přístupu do Windows Forms – ovládací prvky  
  
#### <a name="to-make-a-thread-safe-call-to-a-windows-forms-control"></a>K volání bezpečného přístupu do ovládacího prvku Windows Forms  
  
1.  Dotaz ovládacího prvku <xref:System.Windows.Forms.Control.InvokeRequired%2A> vlastnost.  
  
2.  Pokud <xref:System.Windows.Forms.Control.InvokeRequired%2A> vrátí `true`, volání <xref:System.Windows.Forms.Control.Invoke%2A> s delegáta, který umožňuje vlastní volání do ovládacího prvku.  
  
3.  Pokud <xref:System.Windows.Forms.Control.InvokeRequired%2A> vrátí `false`, ovládacího prvku volat přímo.  
  
 V následujícím příkladu kódu je implementována volání bezpečné pro přístup z více vláken ve `ThreadProcSafe` metodu, která se spustí ve vlákně na pozadí. Pokud <xref:System.Windows.Forms.TextBox> ovládacího prvku <xref:System.Windows.Forms.Control.InvokeRequired%2A> vrátí `true`, `ThreadProcSafe` metoda vytvoří instanci `StringArgReturningVoidDelegate` a který předává do formuláře <xref:System.Windows.Forms.Control.Invoke%2A> metoda. To způsobí, že `SetText` metoda k volání na vlákno, které vytvořili <xref:System.Windows.Forms.TextBox> řízení a v tomto kontextu vláken <xref:System.Windows.Forms.Control.Text%2A> vlastnost je nastavena přímo.  
  
```csharp  
// This event handler creates a thread that calls a   
// Windows Forms control in a thread-safe way.  
private void setTextSafeBtn_Click(  
    object sender,   
    EventArgs e)  
{  
    this.demoThread =   
        new Thread(new ThreadStart(this.ThreadProcSafe));  
  
    this.demoThread.Start();  
}  
  
// This method is executed on the worker thread and makes  
// a thread-safe call on the TextBox control.  
private void ThreadProcSafe()  
{  
    this.SetText("This text was set safely.");  
}  
```

```csharp  
// This method demonstrates a pattern for making thread-safe  
// calls on a Windows Forms control.   
//  
// If the calling thread is different from the thread that  
// created the TextBox control, this method creates a  
// StringArgReturningVoidDelegate and calls itself asynchronously using the  
// Invoke method.  
//  
// If the calling thread is the same as the thread that created  
// the TextBox control, the Text property is set directly.   
  
private void SetText(string text)  
{  
    // InvokeRequired required compares the thread ID of the  
    // calling thread to the thread ID of the creating thread.  
    // If these threads are different, it returns true.  
    if (this.textBox1.InvokeRequired)  
    {     
        StringArgReturningVoidDelegate d = new StringArgReturningVoidDelegate(SetText);  
        this.Invoke(d, new object[] { text });  
    }  
    else  
    {  
        this.textBox1.Text = text;  
    }  
}  
```  
  

## <a name="making-thread-safe-calls-by-using-backgroundworker"></a>BackgroundWorker – pomocí volání bezpečné pro přístup z více vláken  
 Upřednostňovaný způsob implementace více vláken ve vaší aplikaci, je použití <xref:System.ComponentModel.BackgroundWorker> součásti. <xref:System.ComponentModel.BackgroundWorker> Součást používá model založeného na událostech pro více vláken. Vlákně na pozadí běží vaše <xref:System.ComponentModel.BackgroundWorker.DoWork> obslužné rutiny události a vlákno, které vytváří vaše ovládací prvky běží vaše <xref:System.ComponentModel.BackgroundWorker.ProgressChanged> a <xref:System.ComponentModel.BackgroundWorker.RunWorkerCompleted> obslužné rutiny událostí. Můžete volat vaše ovládací prvky z vaší <xref:System.ComponentModel.BackgroundWorker.ProgressChanged> a <xref:System.ComponentModel.BackgroundWorker.RunWorkerCompleted> obslužné rutiny událostí.  
  
#### <a name="to-make-thread-safe-calls-by-using-backgroundworker"></a>BackgroundWorker – pomocí volání bezpečné pro přístup z více vláken  
  
1.  Vytvoření metody, které udělají tuto práci, který chcete provést ve vlákně na pozadí. Ovládací prvky vytvořené pomocí hlavní vlákno v tato metoda není volána.  
  
2.  Vytvořte metodu pro sestavy o výsledcích práce pozadí po jejím dokončení. Ovládací prvky vytvořené pomocí hlavní vlákno v této metody můžete volat.  
  
3.  BIND – metoda vytvořené v kroku 1 <xref:System.ComponentModel.BackgroundWorker.DoWork> události instance <xref:System.ComponentModel.BackgroundWorker>a vytvořte vazbu metodu vytvořili v kroku 2 na stejnou instanci <xref:System.ComponentModel.BackgroundWorker.RunWorkerCompleted> událostí.  
  
4.  Chcete-li spustit vlákně na pozadí, zavolejte <xref:System.ComponentModel.BackgroundWorker.RunWorkerAsync%2A> metodu <xref:System.ComponentModel.BackgroundWorker> instance.  
  
 V následujícím příkladu kódu <xref:System.ComponentModel.BackgroundWorker.DoWork> používá obslužné rutiny události <xref:System.Threading.Thread.Sleep%2A> k simulaci práci, kterou nějakou dobu trvá. Nevyvolá formuláře <xref:System.Windows.Forms.TextBox> ovládacího prvku. <xref:System.Windows.Forms.TextBox> Ovládacího prvku <xref:System.Windows.Forms.Control.Text%2A> přímo v je hodnota nastavena <xref:System.ComponentModel.BackgroundWorker.RunWorkerCompleted> obslužné rutiny události.

## <a name="example"></a>Příklad  
 Následující příklad kódu je kompletní aplikace Windows Forms, která se skládá z formulář, který má tři tlačítka a jedno textové pole. Na první tlačítko ukazuje unsafe přístupu mezi vlákny, druhý tlačítko ukazuje bezpečný přístup pomocí <xref:System.Windows.Forms.Control.Invoke%2A>, a třetí tlačítko ukazuje bezpečný přístup pomocí <xref:System.ComponentModel.BackgroundWorker>.  
  
> [!NOTE]
>  Pokyny o tom, jak spustit v příkladu najdete v tématu [postupy: zkompilování a spuštění dokončení Windows Forms kód příklad pomocí sady Visual Studio](http://msdn.microsoft.com/library/cc447f7e-4c3b-4397-9d05-aeba3ca49416).
