# <a name="net-framework-versions-and-dependencies"></a>Verze a závislosti rozhraní .NET Framework
Všechny verze rozhraní .NET Framework obsahují modul Common Language Runtime (CLR), knihovny základních tříd a další spravované knihovny. Toto téma popisuje klíčové funkce rozhraní .NET Framework podle verzí, poskytuje informace o základních verzích CLR a přidružených vývojových prostředích a identifikuje verze, které jsou nainstalovány v operačním systému Windows.  
  
> [!NOTE]
>  Informace o stahování a instalace rozhraní .NET Framework, najdete v části [instalaci rozhraní .NET Framework pro vývojáře](../../../docs/framework/install/guide-for-developers.md).  
  
 Následující tabulka shrnuje historie verzí rozhraní .NET Framework a korelaci jednotlivých verzí v sadě Visual Studio, Windows a Windows Server. Sada Visual Studio poskytuje možnost zacílení na více verzí, takže nejste omezeni uvedenou verzí rozhraní .NET Framework.  
  
 Každá nová verze rozhraní .NET Framework zachovává funkce předchozích verzí a přidává nové funkce. Modul CLR je identifikován vlastním číslem verze. Číslo verze rozhraní .NET Framework je při každém vydání zvýšeno, i když není vždy zvýšena verze modulu CLR. Pro příklad, rozhraní .NET Framework 4, 4.5 a novějších verzích patří 4 CLR, ale rozhraní .NET Framework 2.0, 3.0 a 3.5 zahrnují CLR 2.0. (Verze 3 CLR neexistovala.)

Ke stažení, najdete v části [instalaci rozhraní .NET Framework pro vývojáře](../../../docs/framework/install/guide-for-developers.md). Určení toho, jaké verze rozhraní .NET Framework se instaluje na počítači, najdete v části [postupy: určení rozhraní .NET Framework verze jsou nainstalované](../../../docs/framework/migration-guide/how-to-determine-which-versions-are-installed.md).

## <a name="targeting-and-running-net-framework-apps-for-version-45-and-later"></a>Aplikace cílené a spuštění rozhraní .NET Framework verze 4.5 nebo novější  
 [!INCLUDE[net_v45](../../../includes/net-v45-md.md)] Je na místě aktualizaci, která nahrazuje [!INCLUDE[net_v40_short](../../../includes/net-v40-short-md.md)] ve vašem počítači a podobně [!INCLUDE[net_v451](../../../includes/net-v451-md.md)] 4.5.2, 4.6, 4.6.1, 4.6.2, 4.7 a 4.7.1 jsou na místě aktualizace [!INCLUDE[net_v45](../../../includes/net-v45-md.md)], což znamená, že používají stejnou verzi modulu runtime ale verze sestavení jsou aktualizovány a zahrnuje nové typy a členy. Po instalaci jedné z těchto aktualizací, vaše [!INCLUDE[net_v40_short](../../../includes/net-v40-short-md.md)], [!INCLUDE[net_v45](../../../includes/net-v45-md.md)], rozhraní .NET Framework 4.6 nebo aplikací rozhraní .NET Framework 4.7 by měly být nadále spustit bez nutnosti rekompilace. Opačně to však neplatí. Nedoporučujeme spouští aplikace cílených na novější verzi rozhraní .NET Framework na dřívější verzi rozhraní .NET Framework. Například není doporučeno spouštět aplikace cíle [!INCLUDE[net_v46](../../../includes/net-v46-md.md)] na [!INCLUDE[net_v45](../../../includes/net-v45-md.md)]. Platí následující pokyny:  
  
-   V sadě Visual Studio, můžete [!INCLUDE[net_v45](../../../includes/net-v45-md.md)] jako cílové rozhraní pro projekt (nastaví tato <xref:Microsoft.Build.Tasks.GetReferenceAssemblyPaths.TargetFrameworkMoniker%2A?displayProperty=nameWithType> vlastnost) ke kompilaci projektu jako [!INCLUDE[net_v45](../../../includes/net-v45-md.md)] sestavení nebo spustitelný soubor. Toto sestavení nebo spustitelný soubor je potom možné na libovolném počítači, který má [!INCLUDE[net_v45](../../../includes/net-v45-md.md)], 4.5.1, 4.5.2, 4.6, 4.6.1, 4.6.2, 4.7 nebo 4.7.1 nainstalována.  
  
-   V sadě Visual Studio, můžete [!INCLUDE[net_v451](../../../includes/net-v451-md.md)] jako cílové rozhraní pro projekt (nastaví tato <xref:Microsoft.Build.Tasks.GetReferenceAssemblyPaths.TargetFrameworkMoniker%2A?displayProperty=nameWithType> vlastnost) ke kompilaci projektu jako [!INCLUDE[net_v451](../../../includes/net-v451-md.md)] sestavení nebo spustitelný soubor. Toto sestavení nebo spustitelný soubor by měl spustit pouze v počítačích, které mají [!INCLUDE[net_v451](../../../includes/net-v451-md.md)] nebo novější verze nainstalováno rozhraní .NET Framework. Spustitelné soubory, které cílí [!INCLUDE[net_v451](../../../includes/net-v451-md.md)] se zablokuje ve spuštění v počítači, který má jenom dřívější verzi rozhraní .NET Framework, jako [!INCLUDE[net_v45](../../../includes/net-v45-md.md)], nainstalovaný, a uživatel bude vyzván k instalaci [!INCLUDE[net_v451](../../../includes/net-v451-md.md)]. Kromě toho [!INCLUDE[net_v451](../../../includes/net-v451-md.md)] sestavení by neměl být volán z aplikace, která cílí na starší verzi rozhraní .NET Framework, jako [!INCLUDE[net_v45](../../../includes/net-v45-md.md)].  
  
     [!INCLUDE[net_v451](../../../includes/net-v451-md.md)] a [!INCLUDE[net_v45](../../../includes/net-v45-md.md)] se používají pouze jako příklady. Tato zásada se použije pro všechny aplikace s cílem novější verzi rozhraní .NET Framework, než požadavek nainstalovaný v systému, na kterém je spuštěný.  
  
 Některé změny v rozhraní .NET Framework může vyžadovat změny kódu aplikace; v tématu [kompatibilita aplikací](../../../docs/framework/migration-guide/application-compatibility.md) před spuštěním vaše stávající aplikace s [!INCLUDE[net_v45](../../../includes/net-v45-md.md)] nebo novější verze. Další informace o instalaci aktuální verzi najdete v tématu [instalaci rozhraní .NET Framework pro vývojáře](../../../docs/framework/install/guide-for-developers.md). Informace o podpoře pro rozhraní .NET Framework najdete v tématu [podporu zásadách životního cyklu Microsoft .NET Framework](http://go.microsoft.com/fwlink/?LinkId=196607) na webu Microsoft Support.  
  
## <a name="targeting-and-running-apps-for-older-versions"></a>Zacílení a spuštění aplikací pro starší verze  
 Verze rozhraní .NET Framework 2.0, 3.0 a 3.5 jsou vybaveny stejnou verzí modulu CLR (CLR 2.0). Tyto verze představují sousední vrstvy jedné instalace. Každá verze je postupně sestavena nad předchozí verzí. Na jednom počítači není možné spouštět verze 2.0, 3.0 a 3.5 současně. Při instalaci verze 3.5 automaticky získáte vrstvy 2.0 a 3.0 a aplikace, které byly vytvořeny pro verze 2.0, 3.0 a 3.5, lze spustit na verzi 3.5. Rozhraní .NET Framework 4, ale ukončí tento rozvrstvení přístup. Počínaje rozhraním .NET Framework 4 můžete použít souběžné hostování v rámci procesu ke spuštění několika verzí modulu CLR v jednom procesu. Další informace najdete v tématu [sestavení a spuštění vedle sebe](../../../docs/framework/app-domains/assemblies-and-side-by-side-execution.md).  
  
 Kromě toho, pokud aplikace cílí verze 2.0, 3.0 nebo 3.5, uživatelé může být nutné povolit v rozhraní .NET Framework 3.5 [!INCLUDE[win8](../../../includes/win8-md.md)] nebo [!INCLUDE[win81](../../../includes/win81-md.md)] počítač ještě před jejich spuštěním aplikace. Další informace najdete v tématu [nainstalovat rozhraní .NET Framework 3.5 ve Windows 10, Windows 8.1 a Windows 8](../../../docs/framework/install/dotnet-35-windows-10.md).
