<span data-ttu-id="cff08-103">Zpětná kompatibilita znamená, že aplikaci, která byla vyvinuta pro konkrétní verze platformy bude spuštěna v novějších verzích této platformě.</span><span class="sxs-lookup"><span data-stu-id="cff08-103">Backward compatibility means that an app that was developed for a particular version of a platform will run on later versions of that platform.</span></span> <span data-ttu-id="cff08-104">Rozhraní .NET Framework snaží co zpětné kompatibility: zdrojový kód napsaný pro jednu verzi rozhraní .NET Framework by měl kompilovat v novějších verzích rozhraní .NET Framework a binární soubory, které běží na jednu verzi rozhraní .NET Framework se měl chovat stejně jako na novější verze rozhraní .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="cff08-104">The .NET Framework tries to maximize backward compatibility: Source code written for one version of the .NET Framework should compile on later versions of the .NET Framework, and binaries that run on one version of the .NET Framework should behave identically on later versions of the .NET Framework.</span></span>

 <span data-ttu-id="cff08-109">K definování konkrétních verzí, na kterých běží vaše aplikace, přidejte jeden nebo více [ \<supportedRuntime >](../../../docs/framework/configure-apps/file-schema/startup/supportedruntime-element.md) elementy do vaší aplikace konfiguračního souboru.</span><span class="sxs-lookup"><span data-stu-id="cff08-109">To define the specific versions on which your app runs, add one or more [\<supportedRuntime>](../../../docs/framework/configure-apps/file-schema/startup/supportedruntime-element.md) elements to your app's configuration file.</span></span> <span data-ttu-id="cff08-110">Každý `<supportedRuntime>` element uvádí podporovanou verzi modulu runtime s prvním zadání nejvíc preferovaný verze a poslední zadání nejméně upřednostňovanou verzi.</span><span class="sxs-lookup"><span data-stu-id="cff08-110">Each `<supportedRuntime>` element lists a supported version of the runtime, with the first specifying the most preferred version and the last specifying the least preferred version.</span></span>

   <startup>  
      <supportedRuntime version="v2.0.50727" />  
      <supportedRuntime version="v4.0" />  
   </startup>  
</configuration>
