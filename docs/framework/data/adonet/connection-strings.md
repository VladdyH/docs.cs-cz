# <a name="connection-strings-in-adonet"></a><span data-ttu-id="a6218-102">Připojovací řetězce v ADO.NET</span><span class="sxs-lookup"><span data-stu-id="a6218-102">Connection Strings in ADO.NET</span></span>
<span data-ttu-id="a6218-103">Rozhraní .NET Framework 2.0 zavedeny nové funkce pro práci s řetězci připojení, včetně představení nových klíčových slov na tvůrce třídy řetězec připojení, které usnadňují vytváření platný připojovací řetězce v době běhu.</span><span class="sxs-lookup"><span data-stu-id="a6218-103">The .NET Framework 2.0 introduced new capabilities for working with connection strings, including the introduction of new keywords to the connection string builder classes, which facilitate creating valid connection strings at run time.</span></span>  
  
 <span data-ttu-id="a6218-104">Připojovací řetězec obsahuje informace o inicializaci, která je předána jako parametr od poskytovatele dat ke zdroji dat.</span><span class="sxs-lookup"><span data-stu-id="a6218-104">A connection string contains initialization information that is passed as a parameter from a data provider to a data source.</span></span> <span data-ttu-id="a6218-105">Syntaxe závisí na poskytovateli dat a připojovací řetězec je analyzován při pokusu o otevření připojení.</span><span class="sxs-lookup"><span data-stu-id="a6218-105">The syntax depends on the data provider, and the connection string is parsed during the attempt to open a connection.</span></span> <span data-ttu-id="a6218-106">Chyby syntaxe vygenerovat výjimku za běhu, ale až po informace o připojení obdrží zdroj dat dojít k jiným chybám.</span><span class="sxs-lookup"><span data-stu-id="a6218-106">Syntax errors generate a run-time exception, but other errors occur only after the data source receives connection information.</span></span> <span data-ttu-id="a6218-107">Jakmile ověříte, zdroj dat platí volby zadané v připojovacím řetězci a otevře připojení.</span><span class="sxs-lookup"><span data-stu-id="a6218-107">Once validated, the data source applies the options specified in the connection string and opens the connection.</span></span>  
  
 <span data-ttu-id="a6218-108">Formát připojovacího řetězce je středníkem oddělený seznam dvojic klíč/hodnota parametru:</span><span class="sxs-lookup"><span data-stu-id="a6218-108">The format of a connection string is a semicolon-delimited list of key/value parameter pairs:</span></span>  
  
 `keyword1=value; keyword2=value;`  
  
 <span data-ttu-id="a6218-109">Klíčová slova nejsou velká a malá písmena a mezery mezi páry klíč/hodnota jsou ignorovány.</span><span class="sxs-lookup"><span data-stu-id="a6218-109">Keywords are not case sensitive, and spaces between key/value pairs are ignored.</span></span> <span data-ttu-id="a6218-110">Hodnoty mohou být velká a malá písmena, v závislosti na zdroji dat.</span><span class="sxs-lookup"><span data-stu-id="a6218-110">However, values may be case sensitive, depending on the data source.</span></span> <span data-ttu-id="a6218-111">Všechny hodnoty obsahující středník, jednoduchých uvozovek nebo dvojité uvozovky musí být uzavřena v dvojitých uvozovkách.</span><span class="sxs-lookup"><span data-stu-id="a6218-111">Any values containing a semicolon, single quotation marks, or double quotation marks must be enclosed in double quotation marks.</span></span>  
  
 <span data-ttu-id="a6218-112">Platný připojovací řetězec syntaxe závisí na poskytovateli a v průběhu let ze starší rozhraní API, jako je rozhraní ODBC se vyvinula.</span><span class="sxs-lookup"><span data-stu-id="a6218-112">Valid connection string syntax depends on the provider, and has evolved over the years from earlier APIs like ODBC.</span></span> <span data-ttu-id="a6218-113">Zprostředkovatel dat .NET Framework pro SQL Server (SqlClient) zahrnuje mnoho prvků ze starší syntaxi a je obecně flexibilnější s běžné syntaxe připojovacího řetězce.</span><span class="sxs-lookup"><span data-stu-id="a6218-113">The .NET Framework Data Provider for SQL Server (SqlClient) incorporates many elements from older syntax and is generally more flexible with common connection string syntax.</span></span> <span data-ttu-id="a6218-114">Jsou často stejně platná synonyma pro elementy syntaxe připojovacího řetězce, ale některé syntaxe a pravopisné chyby může způsobit problémy.</span><span class="sxs-lookup"><span data-stu-id="a6218-114">There are frequently equally valid synonyms for connection string syntax elements, but some syntax and spelling errors can cause problems.</span></span> <span data-ttu-id="a6218-115">Například "`Integrated Security=true`" platí, že "`IntegratedSecurity=true`" způsobí chybu.</span><span class="sxs-lookup"><span data-stu-id="a6218-115">For example, "`Integrated Security=true`" is valid, whereas "`IntegratedSecurity=true`" causes an error.</span></span> <span data-ttu-id="a6218-116">Kromě toho připojovací řetězce vytvořeným za běhu z neověřený vstup uživatele může vést k útokům prostřednictvím injektáže řetězec, ohrožující zabezpečení ve zdroji dat.</span><span class="sxs-lookup"><span data-stu-id="a6218-116">In addition, connection strings constructed at run time from unvalidated user input can lead to string injection attacks, jeopardizing security at the data source.</span></span>  
  
 <span data-ttu-id="a6218-117">K řešení těchto problémů, ADO.NET 2.0 zavedeny nové tvůrci připojovacích řetězců pro každého poskytovatele dat .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="a6218-117">To address these problems, ADO.NET 2.0 introduced new connection string builders for each .NET Framework data provider.</span></span> <span data-ttu-id="a6218-118">Klíčová slova jsou vystaveny jako vlastnosti, povolení syntaxe připojovacího řetězce k ověření před odesláním do zdroje dat.</span><span class="sxs-lookup"><span data-stu-id="a6218-118">Keywords are exposed as properties, enabling connection string syntax to be validated before submission to the data source.</span></span>
