## <a name="wpf-and-assembly-loading"></a><span data-ttu-id="86110-162">WPF a načtení sestavení</span><span class="sxs-lookup"><span data-stu-id="86110-162">WPF and Assembly Loading</span></span>  
 <span data-ttu-id="86110-163">Kontext WPF schématu XAML integruje s modelem aplikace WPF, které dále používá koncept definované CLR <xref:System.AppDomain>.</span><span class="sxs-lookup"><span data-stu-id="86110-163">The XAML schema context for WPF integrates with the WPF application model, which in turn uses the CLR-defined concept of <xref:System.AppDomain>.</span></span> <span data-ttu-id="86110-164">Následující text popisuje, jak interpretovat kontext schématu XAML jak načíst sestavení nebo najít typy v době běhu nebo návrhu, založené na použití WPF <xref:System.AppDomain> a dalších faktorů.</span><span class="sxs-lookup"><span data-stu-id="86110-164">The following sequence describes how XAML schema context interprets how to either load assemblies or find types at run time or design time, based on the WPF use of <xref:System.AppDomain> and other factors.</span></span>  
  
1.  <span data-ttu-id="86110-165">Iterace <xref:System.AppDomain>, hledá již načíst sestavení, která odpovídají všechny aspekty název, od naposledy načíst sestavení.</span><span class="sxs-lookup"><span data-stu-id="86110-165">Iterate through the <xref:System.AppDomain>, looking for an already-loaded assembly that matches all aspects of the name, starting from the most recently loaded assembly.</span></span>  
  
2.  <span data-ttu-id="86110-166">Pokud je kvalifikovaný název, volání <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> na kvalifikovaný název.</span><span class="sxs-lookup"><span data-stu-id="86110-166">If the name is qualified, call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> on the qualified name.</span></span>  
  
3.  <span data-ttu-id="86110-167">Pokud krátký název + token veřejného klíče kvalifikovaný název odpovídá sestavení, které kód byl načten z, vrátí této položky assembly.</span><span class="sxs-lookup"><span data-stu-id="86110-167">If the short name + public key token of a qualified name matches the assembly that the markup was loaded from, return that assembly.</span></span>  
  
4.  <span data-ttu-id="86110-168">Použít krátký název + tokenu veřejného klíče pro volání <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="86110-168">Use the short name + public key token to call <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType>.</span></span>  
  
5.  <span data-ttu-id="86110-169">Pokud neúplný název, volání <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="86110-169">If the name is unqualified, call <xref:System.Reflection.Assembly.LoadWithPartialName%2A?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="86110-170">Přijít XAML nepoužívá krok 3; neexistuje žádné sestavení načíst z.</span><span class="sxs-lookup"><span data-stu-id="86110-170">Loose XAML does not use Step 3; there is no loaded-from assembly.</span></span>  
  
 <span data-ttu-id="86110-171">Kompilované XAML pro grafický subsystém WPF (vygenerovaný prostřednictvím XamlBuildTask) nepoužívá sestavení již načten z <xref:System.AppDomain> (krok 1).</span><span class="sxs-lookup"><span data-stu-id="86110-171">Compiled XAML for WPF (generated via XamlBuildTask) does not use the already-loaded assemblies from <xref:System.AppDomain> (Step 1).</span></span> <span data-ttu-id="86110-172">Název by měl nikdy neúplné z výstupu XamlBuildTask tak nelze použít krok 5.</span><span class="sxs-lookup"><span data-stu-id="86110-172">Also, the name should never be unqualified from XamlBuildTask output, so Step 5 does not apply.</span></span>  
  
 <span data-ttu-id="86110-173">Kompilované BAML (vygenerovaný prostřednictvím PresentationBuildTask) používá všechny kroky, i když BAML také by neměly obsahovat názvy nekvalifikované sestavení.</span><span class="sxs-lookup"><span data-stu-id="86110-173">Compiled BAML (generated via PresentationBuildTask) uses all steps, although BAML also should not contain unqualified assembly names.</span></span>
