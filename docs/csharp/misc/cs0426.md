# <a name="compiler-error-cs0426"></a><span data-ttu-id="173ac-102">CS0426 chyby kompilátoru</span><span class="sxs-lookup"><span data-stu-id="173ac-102">Compiler Error CS0426</span></span>

 <span data-ttu-id="173ac-104">Tato chyba nastane, když odkazujete typu vnořené v rámci jiného typu, ale není žádná taková vnořené, že typ existuje.</span><span class="sxs-lookup"><span data-stu-id="173ac-104">This error occurs when you reference a type nested within another type, but no such nested type exists.</span></span> <span data-ttu-id="173ac-105">Tato situace může nastat, pokud překlep název vnořené typu.</span><span class="sxs-lookup"><span data-stu-id="173ac-105">This can occur if you mistype the name of the nested type.</span></span> <span data-ttu-id="173ac-106">Zkontrolujte, zda názvů používaných a ověřte, zda očekávaný člen nadřazených typů.</span><span class="sxs-lookup"><span data-stu-id="173ac-106">Check the spelling of the names used, and verify that the enclosing type has the expected member.</span></span>  
  
 <span data-ttu-id="173ac-107">Následující ukázka generuje CS0426, protože třída C nemá žádné vnořené typy A:</span><span class="sxs-lookup"><span data-stu-id="173ac-107">The following sample generates CS0426 because class C has no nested type A:</span></span>  
  
```csharp  
// CS0426.cs  
  
class C  
{  
    // No nested types are declared.     
}  
  
class D  
{  
   public static void Main()  
   {  
       C c = new C();  
       // Attempt to reference a nested type A:  
       C.A a; // CS0426 because no such type C.A  
   }  
}  
```
