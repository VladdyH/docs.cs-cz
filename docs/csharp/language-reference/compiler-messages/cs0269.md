# <a name="compiler-error-cs0269"></a><span data-ttu-id="61264-102">Chyba kompilátoru CS0269</span><span class="sxs-lookup"><span data-stu-id="61264-102">Compiler Error CS0269</span></span>
<span data-ttu-id="61264-103">Použití nepřiřazeného výstupního parametru "parametr"</span><span class="sxs-lookup"><span data-stu-id="61264-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="61264-104">Kompilátor nemůže ověřit, že výstupní parametr byl přiřazenou hodnotu, než byl použit; jeho hodnota může být při přiřazení nedefinovaná.</span><span class="sxs-lookup"><span data-stu-id="61264-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="61264-105">Ujistěte se o přiřazení hodnoty k `out` parametry volané metody před přístupem k hodnotě.</span><span class="sxs-lookup"><span data-stu-id="61264-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="61264-106">Pokud potřebujete použít hodnotu proměnné předaná, použijte `ref` parametr místo.</span><span class="sxs-lookup"><span data-stu-id="61264-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="61264-107">Další informace najdete v tématu [předávání parametry](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="61264-107">For more information, see [Passing Parameters](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="61264-108">Příklad</span><span class="sxs-lookup"><span data-stu-id="61264-108">Example</span></span>  
 <span data-ttu-id="61264-109">Následující ukázka generuje CS0269:</span><span class="sxs-lookup"><span data-stu-id="61264-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="61264-110">Příklad</span><span class="sxs-lookup"><span data-stu-id="61264-110">Example</span></span>  
 <span data-ttu-id="61264-111">Tato situace může nastat v případě inicializace proměnné v bloku try což kompilátor je schopen ověřit spustí úspěšně:</span><span class="sxs-lookup"><span data-stu-id="61264-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
