1. <span data-ttu-id="52101-107">Pokud proměnná rozsahu je explicitně zadán, ujistěte se, že typ je buď stejná jako nebo iterace, je implicitně převést z typu prvků v kolekci.</span><span class="sxs-lookup"><span data-stu-id="52101-107">If the range variable is explicitly typed, make sure that the type is either the same as, or implicitly convertible from, the type of the elements in the collection it iterates.</span></span> <span data-ttu-id="52101-108">Pokud je proměnná rozsahu začínající `var` – klíčové slovo, odeberte `var`.</span><span class="sxs-lookup"><span data-stu-id="52101-108">If the range variable is preceded with the `var` keyword, remove `var`.</span></span>  
  
## <a name="example"></a><span data-ttu-id="52101-109">Příklad</span><span class="sxs-lookup"><span data-stu-id="52101-109">Example</span></span>

<span data-ttu-id="52101-110">Následující kód vygeneruje CS1942:</span><span class="sxs-lookup"><span data-stu-id="52101-110">The following code generates CS1942:</span></span>  

```csharp
// cs1942.cs  
class Program  
    {  
        static void Main(string[] args)  
        {  
            var x = from var i in Enumerable.Range(1, 100) // CS1949  
                    select i; //CS1942  
        }  
    }  
```

<span data-ttu-id="52101-111">Protože souvisí CS1949 CS1942 použití `var` s rozsahem proměnné způsobí, že základní `Cast<T>` operace se nezdařila, protože `var` není typem.</span><span class="sxs-lookup"><span data-stu-id="52101-111">CS1942 is related to CS1949 because the use of `var` with a range variable causes the underlying `Cast<T>` operation to fail because `var` is not a type.</span></span>
