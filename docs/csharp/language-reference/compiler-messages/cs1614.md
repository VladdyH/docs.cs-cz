# <a name="compiler-error-cs1614"></a><span data-ttu-id="c7f00-102">Chyba kompilátoru CS1614</span><span class="sxs-lookup"><span data-stu-id="c7f00-102">Compiler Error CS1614</span></span>
<span data-ttu-id="c7f00-103">"název" je nejednoznačný; mezi "atribut1" a "atribut2".</span><span class="sxs-lookup"><span data-stu-id="c7f00-103">'name' is ambiguous; between 'attribute1' and 'attribute2'.</span></span> <span data-ttu-id="c7f00-104">Použijte buď "@attribute" nebo "atributAtribut"</span><span class="sxs-lookup"><span data-stu-id="c7f00-104">use either '@attribute' or 'attributeAttribute'</span></span>  
  
 <span data-ttu-id="c7f00-105">Kompilátor došlo specifikace nejednoznačný atribut.</span><span class="sxs-lookup"><span data-stu-id="c7f00-105">The compiler has encountered an ambiguous attribute specification.</span></span>  
  
 <span data-ttu-id="c7f00-106">Kompilátor jazyka C# pro usnadnění práce umožňuje určit **ExampleAttribute** jako jenom `[Example]`.</span><span class="sxs-lookup"><span data-stu-id="c7f00-106">For convenience, the C# compiler allows you to specify **ExampleAttribute** as just `[Example]`.</span></span> <span data-ttu-id="c7f00-107">Nejednoznačnost však nastane, pokud název třídy atributu `Example` existuje spolu s **ExampleAttribute**, protože nemůže určit kompilátor Pokud `[Example]` odkazuje `Example` atribut nebo  **ExampleAttribute** atribut.</span><span class="sxs-lookup"><span data-stu-id="c7f00-107">However, ambiguity arises if an attribute class named `Example` exists along with **ExampleAttribute**, because the compiler cannot tell if `[Example]` refers to the `Example` attribute or the **ExampleAttribute** attribute.</span></span> <span data-ttu-id="c7f00-108">O vysvětlení, použijte `[@Example]` pro `Example` atribut a `[ExampleAttribute]` pro **ExampleAttribute**.</span><span class="sxs-lookup"><span data-stu-id="c7f00-108">To clarify, use `[@Example]` for the `Example` attribute and `[ExampleAttribute]` for **ExampleAttribute**.</span></span>  
  
 <span data-ttu-id="c7f00-109">Následující ukázka generuje CS1614:</span><span class="sxs-lookup"><span data-stu-id="c7f00-109">The following sample generates CS1614:</span></span>  
  
```csharp  
// CS1614.cs  
using System;  
  
// Both of the following classes are valid attributes with valid  
// names (MySpecial and MySpecialAttribute). However, because the lookup  
// rules for attributes involves auto-appending the 'Attribute' suffix  
// to the identifier, these two attributes become ambiguous; that is,  
// if you specify MySpecial, the compiler can't tell if you want  
// MySpecial or MySpecialAttribute.  
  
public class MySpecial : Attribute {  
   public MySpecial() {}  
}  
  
public class MySpecialAttribute : Attribute {  
   public MySpecialAttribute() {}  
}  
  
class MakeAWarning {  
   [MySpecial()] // CS1614  
                 // Ambiguous: MySpecial or MySpecialAttribute?  
   public static void Main() {  
   }  
  
   [@MySpecial()] // This isn't ambiguous, it binds to the first attribute above.  
   public static void NoWarning() {  
   }  
  
   [MySpecialAttribute()] // This isn't ambiguous, it binds to the second attribute above.  
   public static void NoWarning2() {  
   }  
  
   [@MySpecialAttribute()] // This is also legal.  
   public static void NoWarning3() {  
   }  
}  
```
