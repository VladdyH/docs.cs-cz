# <a name="compiler-error-cs1612"></a><span data-ttu-id="20861-102">Chyba kompilátoru CS1612</span><span class="sxs-lookup"><span data-stu-id="20861-102">Compiler Error CS1612</span></span>
<span data-ttu-id="20861-103">Návratovou hodnotu "výraz" nelze změnit, protože se nejedná o proměnnou</span><span class="sxs-lookup"><span data-stu-id="20861-103">Cannot modify the return value of 'expression' because it is not a variable</span></span>  
  
 <span data-ttu-id="20861-104">Došlo pokusu o upravit hodnotový typ, který je vytvořen jako výsledek výrazu zprostředkující, ale není uložený v proměnné.</span><span class="sxs-lookup"><span data-stu-id="20861-104">An attempt was made to modify a value type that is produced as the result of an intermediate expression but is not stored in a variable.</span></span> <span data-ttu-id="20861-105">Této chybě může dojít při pokusu o přímo upravit struktura v obecné kolekce, jak je znázorněno v následujícím příkladu:</span><span class="sxs-lookup"><span data-stu-id="20861-105">This error can occur when you attempt to directly modify a struct in a generic collection, as shown in the following example:</span></span>  
  
```csharp  
List<myStruct> list = {…};  
list[0].Name = "MyStruct42"; //CS1612  
```  
  
 <span data-ttu-id="20861-106">Změnit strukturu, nejprve ji přiřadit do místní proměnné, upravte proměnné a potom přiřadit proměnné zpět na položku v kolekci.</span><span class="sxs-lookup"><span data-stu-id="20861-106">To modify the struct, first assign it to a local variable, modify the variable, then assign the variable back to the item in the collection.</span></span>  
  
```csharp  
List<myStruct> list = {…};  
MyStruct ms = list[0];  
ms.Name = "MyStruct42";  
list[0] = ms;  
```  
  
 <span data-ttu-id="20861-107">K této chybě dochází, protože typy hodnot se zkopírují na přiřazení.</span><span class="sxs-lookup"><span data-stu-id="20861-107">This error occurs because value types are copied on assignment.</span></span> <span data-ttu-id="20861-108">Při načítání typu hodnoty z vlastnosti nebo indexeru, získáte kopii objektu, ne odkaz na samotný objekt.</span><span class="sxs-lookup"><span data-stu-id="20861-108">When you retrieve a value type from a property or indexer, you are getting a copy of the object, not a reference to the object itself.</span></span> <span data-ttu-id="20861-109">Kopírování, která je vrácena neukládá vlastnost nebo indexovací člen vzhledem k tomu, že jsou ve skutečnosti metody, ne umístěními úložiště (proměnné).</span><span class="sxs-lookup"><span data-stu-id="20861-109">The copy that is returned is not stored by the property or indexer because they are actually methods, not storage locations (variables).</span></span> <span data-ttu-id="20861-110">Musíte uložit kopii do proměnné, že deklarujete předtím, než můžete ho upravit.</span><span class="sxs-lookup"><span data-stu-id="20861-110">You must store the copy into a variable that you declare before you can modify it.</span></span>  
  
 <span data-ttu-id="20861-111">Chyba nedojde s typy odkazů, protože vlastnost nebo indexovací člen v takovém případě vrátí odkaz na existující objekt, který je umístěním úložiště.</span><span class="sxs-lookup"><span data-stu-id="20861-111">The error does not occur with reference types because a property or indexer in that case returns a reference to an existing object, which is a storage location.</span></span>  
  
 <span data-ttu-id="20861-112">Pokud definujete třídu nebo strukturu, můžete tuto chybu vyřešit změnou deklarace vlastnosti k poskytnutí přístupu k členům struktury.</span><span class="sxs-lookup"><span data-stu-id="20861-112">If you are defining the class or struct, you can resolve this error by modifying your property declaration to provide access to the members of a struct.</span></span> <span data-ttu-id="20861-113">Pokud píšete kód klienta, lze vyřešit chybu vytvoření vlastní instance struktury, úprava polí a přiřazením celé struktury zpět na vlastnost.</span><span class="sxs-lookup"><span data-stu-id="20861-113">If you are writing client code, you can resolve the error by creating your own instance of the struct, modifying its fields, and then assigning the entire struct back to the property.</span></span> <span data-ttu-id="20861-114">Jako třetí možnost můžete změnit vaše struktury na třídu.</span><span class="sxs-lookup"><span data-stu-id="20861-114">As a third alternative, you can change your struct to a class.</span></span>  
  
## <a name="example"></a><span data-ttu-id="20861-115">Příklad</span><span class="sxs-lookup"><span data-stu-id="20861-115">Example</span></span>  
 <span data-ttu-id="20861-116">CS1612 také vyvolá se při pokusu o přístup k členu struktury přes vlastnost u nadřazené třídy, která vrací celé struktury, jak je znázorněno v následujícím příkladu:</span><span class="sxs-lookup"><span data-stu-id="20861-116">CS1612 also occurs when you attempt to access the member of a struct through a property on an enclosing class that is returning the entire struct, as shown in the following example:</span></span>  
  
```csharp  
// CS1612.cs  
using System;  
  
public struct MyStruct  
{  
    public int Width;  
}  
  
public class ListView  
{  
    MyStruct ms;  
    public MyStruct Size  
    {  
        get { return ms; }  
        set { ms = value; }  
    }  
}  
  
public class MyClass  
{  
    public MyClass()  
    {  
        ListView lvi;  
        lvi = new ListView();  
        lvi.Size.Width = 5; // CS1612  
  
        // You can use the following lines instead.  
        // MyStruct ms;  
        // ms.Width = 5;  
        // lvi.Size = ms;  // CS1612  
    }  
  
    public static void Main()   
    {  
        MyClass mc = new MyClass();  
        // Keep the console open in debug mode.  
        Console.WriteLine("Press any key to exit.");  
        Console.ReadKey();     
    }  
}  
```
