<span data-ttu-id="de234-103">Nejednoznačnost mezi metoda' a jiné metody "jiné metody". Pomocí skupiny metoda.</span><span class="sxs-lookup"><span data-stu-id="de234-103">Ambiguity between method 'method' and non-method 'non-method'.Using method group.</span></span>  
  
 <span data-ttu-id="de234-104">Zděděné členy z různých rozhraní, které mají stejný podpis způsobit chybu nejednoznačnosti.</span><span class="sxs-lookup"><span data-stu-id="de234-104">Inherited members from different interfaces that have the same signature  cause an ambiguity error.</span></span>  
  
## <a name="example"></a><span data-ttu-id="de234-105">Příklad</span><span class="sxs-lookup"><span data-stu-id="de234-105">Example</span></span>  
 <span data-ttu-id="de234-106">Následující příklad generuje CS0467.</span><span class="sxs-lookup"><span data-stu-id="de234-106">The following example generates CS0467.</span></span>  
  
```csharp  
// CS0467.cs  
interface IList   
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver   
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);     
        // To resolve the warning, you can change the name of the method or   
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()   
    {  
    }  
}  
```
