<span data-ttu-id="6bd10-103">Práce s vlákny umožňuje programu jazyka C# provádět souběžné zpracování tak, že máte více než jednu operaci najednou.</span><span class="sxs-lookup"><span data-stu-id="6bd10-103">Threading enables your C# program to perform concurrent processing so that you can do more than one operation at a time.</span></span> <span data-ttu-id="6bd10-104">Například můžete dělení na vlákna pro monitorování vstup od uživatele, provedení úlohy na pozadí a zpracování současných vstupů.</span><span class="sxs-lookup"><span data-stu-id="6bd10-104">For example, you can use threading to monitor input from the user, perform background tasks, and handle simultaneous streams of input.</span></span>  
  
 <span data-ttu-id="6bd10-105">Vlákna mají následující vlastnosti:</span><span class="sxs-lookup"><span data-stu-id="6bd10-105">Threads have the following properties:</span></span>  
  
-   <span data-ttu-id="6bd10-106">Vlákna povolit program provádět souběžné zpracování.</span><span class="sxs-lookup"><span data-stu-id="6bd10-106">Threads enable your program to perform concurrent processing.</span></span>

 <span data-ttu-id="6bd10-110">Ve výchozím nastavení programu v jazyce C# má jedno vlákno.</span><span class="sxs-lookup"><span data-stu-id="6bd10-110">By default, a C# program has one thread.</span></span> <span data-ttu-id="6bd10-111">Pomocná vlákna však můžete vytvořit a použít ke spouštění kódu vytvořeného souběžně s primární vlákno.</span><span class="sxs-lookup"><span data-stu-id="6bd10-111">However, auxiliary threads can be created and used to execute code in parallel with the primary thread.</span></span> <span data-ttu-id="6bd10-112">Tato vlákna se často nazývají *pracovních vláken*.</span><span class="sxs-lookup"><span data-stu-id="6bd10-112">These threads are often called *worker threads*.</span></span>  
  
 <span data-ttu-id="6bd10-113">Pracovní vlákna je možné provádět časově náročné nebo kritického pro čas úlohy bez obsadit primárnímu vláknu.</span><span class="sxs-lookup"><span data-stu-id="6bd10-113">Worker threads can be used to perform time-consuming or time-critical tasks without tying up the primary thread.</span></span> <span data-ttu-id="6bd10-114">Například pracovní vlákna se často používají v serverových aplikacích ke splnění požadavků na příchozí bez čekání na dokončení předchozí žádosti.</span><span class="sxs-lookup"><span data-stu-id="6bd10-114">For example, worker threads are often used in server applications to fulfill incoming requests without waiting for the previous request to be completed.</span></span> <span data-ttu-id="6bd10-115">Pracovní vlákna se taky používají k provádění úloh "pozadí" v desktopových aplikací tak, aby hlavního vlákna – která určuje prvky uživatelského rozhraní – zůstane reagovat na akce uživatele.</span><span class="sxs-lookup"><span data-stu-id="6bd10-115">Worker threads are also used to perform "background" tasks in desktop applications so that the main thread--which drives user interface elements--remains responsive to user actions.</span></span>  
  
 <span data-ttu-id="6bd10-116">Práce s vlákny řeší problémy s propustností a rychlostí odezvy, ale můžou také zavádět sdílení prostředků problémů například zablokování a konflikty časování.</span><span class="sxs-lookup"><span data-stu-id="6bd10-116">Threading solves problems with throughput and responsiveness, but it can also introduce resource-sharing issues such as deadlocks and race conditions.</span></span> <span data-ttu-id="6bd10-117">Více vláken jsou nejvhodnější pro úlohy, které vyžadují různé prostředky, jako jsou popisovače souborů a připojení k síti.</span><span class="sxs-lookup"><span data-stu-id="6bd10-117">Multiple threads are best for tasks that require different resources such as file handles and network connections.</span></span> <span data-ttu-id="6bd10-118">Přiřazení většího počtu vláken na jeden prostředek je pravděpodobně způsobí problémy synchronizace a s často blokované čekající na jiná vlákna vlákna poráží účel používání více vláken.</span><span class="sxs-lookup"><span data-stu-id="6bd10-118">Assigning multiple threads to a single resource is likely to cause synchronization issues, and having threads frequently blocked when waiting for other threads defeats the purpose of using multiple threads.</span></span>  
  
 <span data-ttu-id="6bd10-119">Běžné strategie je použití pracovních vláken provádět časově náročné nebo náročné úkoly, které nevyžadují řadu prostředky používané jinými vlákny.</span><span class="sxs-lookup"><span data-stu-id="6bd10-119">A common strategy is to use worker threads to perform time-consuming or time-critical tasks that do not require many of the resources used by other threads.</span></span> <span data-ttu-id="6bd10-120">Některé prostředky ve svém programu musí být samozřejmě přístup prostřednictvím více vláken.</span><span class="sxs-lookup"><span data-stu-id="6bd10-120">Naturally, some resources in your program must be accessed by multiple threads.</span></span> <span data-ttu-id="6bd10-121">Pro tyto případy <xref:System.Threading> obor názvů obsahuje třídy pro synchronizaci vláken.</span><span class="sxs-lookup"><span data-stu-id="6bd10-121">For these cases, the <xref:System.Threading> namespace provides classes for synchronizing threads.</span></span> <span data-ttu-id="6bd10-122">Tyto třídy zahrnují <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, a <xref:System.Threading.ManualResetEvent>.</span><span class="sxs-lookup"><span data-stu-id="6bd10-122">These classes include <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, and <xref:System.Threading.ManualResetEvent>.</span></span>  
  
 <span data-ttu-id="6bd10-123">Některé nebo všechny tyto třídy můžete použít pro synchronizaci činností více vláken, ale některé podpora pro dělení na vlákna je podporována v jazyce C#.</span><span class="sxs-lookup"><span data-stu-id="6bd10-123">You can use some or all these classes to synchronize the activities of multiple threads, but some support for threading is supported by the C# language.</span></span> <span data-ttu-id="6bd10-124">Například [příkaz Lock](../../../../csharp/language-reference/keywords/lock-statement.md) poskytuje funkce synchronizace prostřednictvím implicitního použití <xref:System.Threading.Monitor>.</span><span class="sxs-lookup"><span data-stu-id="6bd10-124">For example, the [Lock Statement](../../../../csharp/language-reference/keywords/lock-statement.md) provides synchronization features through implicit use of <xref:System.Threading.Monitor>.</span></span>
