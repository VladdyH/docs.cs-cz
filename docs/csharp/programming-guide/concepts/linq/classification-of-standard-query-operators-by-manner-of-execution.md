## <a name="manners-of-execution"></a><span data-ttu-id="4ada6-108">Způsobů spuštění</span><span class="sxs-lookup"><span data-stu-id="4ada6-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="4ada6-109">Okamžitou</span><span class="sxs-lookup"><span data-stu-id="4ada6-109">Immediate</span></span>  
 <span data-ttu-id="4ada6-110">Okamžité spuštění znamená, že zdroj dat je pro čtení a operace proběhne v bodě v kódu, kde je deklarovaná dotazu.</span><span class="sxs-lookup"><span data-stu-id="4ada6-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="4ada6-111">Všechny operátory standardní dotazu, které vrácení výsledku jeden, vyčíslitelná spustit okamžitě.</span><span class="sxs-lookup"><span data-stu-id="4ada6-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="4ada6-112">Odložené</span><span class="sxs-lookup"><span data-stu-id="4ada6-112">Deferred</span></span>  
 <span data-ttu-id="4ada6-113">Odložené provedení znamená, že operaci není provést v bodě v kódu, kde je deklarovaná dotaz.</span><span class="sxs-lookup"><span data-stu-id="4ada6-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="4ada6-114">Operaci se provádí pouze v případě, že proměnná dotazu uvedené, například pomocí `foreach` příkaz.</span><span class="sxs-lookup"><span data-stu-id="4ada6-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="4ada6-115">To znamená, že výsledky zpracování dotazu závisí na obsahu zdroje dat, když je místo, když je definována dotaz spustit dotaz.</span><span class="sxs-lookup"><span data-stu-id="4ada6-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="4ada6-116">Pokud proměnná dotazu je výčet vícekrát, výsledky mohou lišit pokaždé, když.</span><span class="sxs-lookup"><span data-stu-id="4ada6-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="4ada6-117">Téměř všechny standardní operátory dotazu s návratovým typem <xref:System.Collections.Generic.IEnumerable%601> nebo <xref:System.Linq.IOrderedEnumerable%601> provést odložené způsobem.</span><span class="sxs-lookup"><span data-stu-id="4ada6-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="4ada6-118">Operátory dotazu, které používají odložené provedení můžete dále jsou klasifikovány jako streamování nebo jiných datových proudů.</span><span class="sxs-lookup"><span data-stu-id="4ada6-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="4ada6-119">streamování</span><span class="sxs-lookup"><span data-stu-id="4ada6-119">Streaming</span></span>  
 <span data-ttu-id="4ada6-120">Streamování operátory nemusíte před jejich získání prvků si přečtěte všechna zdrojová data.</span><span class="sxs-lookup"><span data-stu-id="4ada6-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="4ada6-121">V době spuštění operátor streamování provede jeho operace pro každý zdroj element tak, jak je pro čtení a vypočítá prvku podle potřeby.</span><span class="sxs-lookup"><span data-stu-id="4ada6-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="4ada6-122">Operátor streamování pokračovat ve čtení zdrojové elementy, dokud je možné vytvořit element výsledek.</span><span class="sxs-lookup"><span data-stu-id="4ada6-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="4ada6-123">To znamená, že může být více než jeden element source pro vytvoření jeden element výsledek ke čtení.</span><span class="sxs-lookup"><span data-stu-id="4ada6-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="4ada6-124">Streamování</span><span class="sxs-lookup"><span data-stu-id="4ada6-124">Non-Streaming</span></span>  
 <span data-ttu-id="4ada6-125">Non-operátory pro datový proud musí čtení všechna zdrojová data, před dávají element výsledek.</span><span class="sxs-lookup"><span data-stu-id="4ada6-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="4ada6-126">Do této kategorie patří operací, jako je řazení nebo seskupování.</span><span class="sxs-lookup"><span data-stu-id="4ada6-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="4ada6-127">V době spuštění-streamování operátory dotazu číst všechna zdrojová data, umístí jej do datová struktura, operaci provést a získání výsledné prvků.</span><span class="sxs-lookup"><span data-stu-id="4ada6-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
