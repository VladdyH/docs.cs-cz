## <a name="remarks"></a><span data-ttu-id="92e83-128">Poznámky</span><span class="sxs-lookup"><span data-stu-id="92e83-128">Remarks</span></span>  
 <span data-ttu-id="92e83-129">Anonymní typy jsou [třída](../../../csharp/language-reference/keywords/class.md) typy, které jsou odvozeny přímo z [objekt](../../../csharp/language-reference/keywords/object.md), a který nelze převést na libovolný typ kromě [objekt](../../../csharp/language-reference/keywords/object.md).</span><span class="sxs-lookup"><span data-stu-id="92e83-129">Anonymous types are [class](../../../csharp/language-reference/keywords/class.md) types that derive directly from [object](../../../csharp/language-reference/keywords/object.md), and that cannot be cast to any type except [object](../../../csharp/language-reference/keywords/object.md).</span></span> <span data-ttu-id="92e83-130">Kompilátor poskytuje název pro každý anonymní typ, i když k nim nemá přístup vaší aplikace.</span><span class="sxs-lookup"><span data-stu-id="92e83-130">The compiler provides a name for each anonymous type, although your application cannot access it.</span></span> <span data-ttu-id="92e83-131">Z pohledu modul common language runtime anonymní typ se neliší od jiných odkazového typu.</span><span class="sxs-lookup"><span data-stu-id="92e83-131">From the perspective of the common language runtime, an anonymous type is no different from any other reference type.</span></span>  
  
 <span data-ttu-id="92e83-132">Pokud dva nebo více inicializátory anonymní objekt v sestavení zadat posloupnost vlastnosti, které jsou ve stejném pořadí a které mají stejné názvy a typy, kompilátor objekty považuje za instance stejného typu.</span><span class="sxs-lookup"><span data-stu-id="92e83-132">If two or more anonymous object initializers in an assembly specify a sequence of properties that are in the same order and that have the same names and types, the compiler treats the objects as instances of the same type.</span></span> <span data-ttu-id="92e83-133">Budou sdílet stejné informace generované kompilátorem typu.</span><span class="sxs-lookup"><span data-stu-id="92e83-133">They share the same compiler-generated type information.</span></span>  
  
 <span data-ttu-id="92e83-134">Pole, vlastnost, událost nebo návratový typ metody tak, že má anonymní typ nelze deklarovat.</span><span class="sxs-lookup"><span data-stu-id="92e83-134">You cannot declare a field, a property, an event, or the return type of a method as having an anonymous type.</span></span> <span data-ttu-id="92e83-135">Podobně nelze deklarovat formální parametr metoda, vlastnost, konstruktor nebo indexer tak, že má anonymního typu.</span><span class="sxs-lookup"><span data-stu-id="92e83-135">Similarly, you cannot declare a formal parameter of a method, property, constructor, or indexer as having an anonymous type.</span></span> <span data-ttu-id="92e83-136">Předávání anonymní typ nebo kolekce, která obsahuje anonymní typy jako argument pro metodu, můžou deklarovat parametr jako typ objektu.</span><span class="sxs-lookup"><span data-stu-id="92e83-136">To pass an anonymous type, or a collection that contains anonymous types, as an argument to a method, you can declare the parameter as type object.</span></span> <span data-ttu-id="92e83-137">To však účinně chrání před účelem silného typování.</span><span class="sxs-lookup"><span data-stu-id="92e83-137">However, doing this defeats the purpose of strong typing.</span></span> <span data-ttu-id="92e83-138">Pokud musí ukládání výsledků dotazu nebo předejte je mimo hranice metoda, zvažte použití obyčejnou struktura s názvem nebo třída místo anonymního typu.</span><span class="sxs-lookup"><span data-stu-id="92e83-138">If you must store query results or pass them outside the method boundary, consider using an ordinary named struct or class instead of an anonymous type.</span></span>  
  
 <span data-ttu-id="92e83-139">Protože <xref:System.Object.Equals%2A> a <xref:System.Object.GetHashCode%2A> metody pro anonymní typy jsou definovány z hlediska `Equals` a `GetHashCode` vlastností dvě instance stejného typu anonymní metody jsou stejné, pouze v případě, že jejich vlastnosti jsou stejné.</span><span class="sxs-lookup"><span data-stu-id="92e83-139">Because the <xref:System.Object.Equals%2A> and <xref:System.Object.GetHashCode%2A> methods on anonymous types are defined in terms of the `Equals` and `GetHashCode` methods of the properties, two instances of the same anonymous type are equal only if all their properties are equal.</span></span>
