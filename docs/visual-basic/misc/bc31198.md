<span data-ttu-id="67c3b-103">Deklarace literálů XML se používá ve výrazu v umístění, které je nejednoznačné k [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] kompilátoru.</span><span class="sxs-lookup"><span data-stu-id="67c3b-103">An XML literal declaration is used in an expression in a location that is ambiguous to the [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] compiler.</span></span> <span data-ttu-id="67c3b-104">To znamená [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] kompilátoru nemůže určit, zda je menší – než znak (<) je určený jako srovnávací operátor nebo spuštění literál XML.</span><span class="sxs-lookup"><span data-stu-id="67c3b-104">That is, the [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] compiler cannot determine whether the less-than character (<) is intended as a comparative operator or the start of an XML literal.</span></span> <span data-ttu-id="67c3b-105">Následující kód představuje příklad:</span><span class="sxs-lookup"><span data-stu-id="67c3b-105">The following code provides an example:</span></span>
