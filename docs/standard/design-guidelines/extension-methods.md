# <a name="extension-methods"></a><span data-ttu-id="124e8-102">Metody rozšíření</span><span class="sxs-lookup"><span data-stu-id="124e8-102">Extension Methods</span></span>
<span data-ttu-id="124e8-103">Rozšiřující metody jsou jazyk funkce, která umožňuje statické metody, která se má volat pomocí volání syntaxi metody instance.</span><span class="sxs-lookup"><span data-stu-id="124e8-103">Extension methods are a language feature that allows static methods to be called using instance method call syntax.</span></span> <span data-ttu-id="124e8-104">Tyto metody vyžaduje minimálně jeden parametr, který představuje instanci, kterou je metoda má být použito.</span><span class="sxs-lookup"><span data-stu-id="124e8-104">These methods must take at least one parameter, which represents the instance the method is to operate on.</span></span>  
  
 <span data-ttu-id="124e8-105">Třídu, která definuje tyto rozšiřující metody se označuje jako "sponzor" třída a ho musí být deklarována jako statické.</span><span class="sxs-lookup"><span data-stu-id="124e8-105">The class that defines such extension methods is referred to as the "sponsor" class, and it must be declared as static.</span></span> <span data-ttu-id="124e8-106">Použití metod rozšíření, jeden musíte importovat definice třídy sponzor obor názvů.</span><span class="sxs-lookup"><span data-stu-id="124e8-106">To use extension methods, one must import the namespace defining the sponsor class.</span></span>  
  
 <span data-ttu-id="124e8-107">**X nepoužívejte** frivolously definování rozšiřující metody, hlavně na typech nevlastníte.</span><span class="sxs-lookup"><span data-stu-id="124e8-107">**X AVOID** frivolously defining extension methods, especially on types you don’t own.</span></span>  
  
 <span data-ttu-id="124e8-108">Pokud jste vlastníkem zdrojového kódu typu, zvažte použití metody normální instanci.</span><span class="sxs-lookup"><span data-stu-id="124e8-108">If you do own source code of a type, consider using regular instance methods instead.</span></span> <span data-ttu-id="124e8-109">Pokud nevlastníte a které chcete přidat metodu, buďte velmi opatrní.</span><span class="sxs-lookup"><span data-stu-id="124e8-109">If you don’t own, and you want to add a method, be very careful.</span></span> <span data-ttu-id="124e8-110">Volná použití metody rozšíření má potenciál zbytečného rozhraní API typů, které nebyly navrženy tak, aby tyto metody.</span><span class="sxs-lookup"><span data-stu-id="124e8-110">Liberal use of extension methods has the potential of cluttering APIs of types that were not designed to have these methods.</span></span>  
  
 <span data-ttu-id="124e8-111">**✓ ZVAŽTE** metodami rozšíření v některém z následujících scénářů:</span><span class="sxs-lookup"><span data-stu-id="124e8-111">**✓ CONSIDER** using extension methods in any of the following scenarios:</span></span>  
  
-   <span data-ttu-id="124e8-112">Zajistit pomocné funkce, které jsou relevantní pro každou implementaci rozhraní, pokud uvedená funkce může být napsán z hlediska základní rozhraní.</span><span class="sxs-lookup"><span data-stu-id="124e8-112">To provide helper functionality relevant to every implementation of an interface, if said functionality can be written in terms of the core interface.</span></span> <span data-ttu-id="124e8-113">Je to proto, že konkrétní implementace v opačném případě nemůže být přiřazen rozhraní.</span><span class="sxs-lookup"><span data-stu-id="124e8-113">This is because concrete implementations cannot otherwise be assigned to interfaces.</span></span> <span data-ttu-id="124e8-114">Například `LINQ to Objects` operátory jsou implementované jako rozšiřující metody pro všechny <xref:System.Collections.Generic.IEnumerable%601> typy.</span><span class="sxs-lookup"><span data-stu-id="124e8-114">For example, the `LINQ to Objects` operators are implemented as extension methods for all <xref:System.Collections.Generic.IEnumerable%601> types.</span></span> <span data-ttu-id="124e8-115">Proto všechny `IEnumerable<>` implementace je automaticky povolené LINQ.</span><span class="sxs-lookup"><span data-stu-id="124e8-115">Thus, any `IEnumerable<>` implementation is automatically LINQ-enabled.</span></span>  
  
-   <span data-ttu-id="124e8-116">Když metoda instance by vznikla závislost na nějaký typ, ale tato závislost by došlo k přerušení pravidla správy závislostí.</span><span class="sxs-lookup"><span data-stu-id="124e8-116">When an instance method would introduce a dependency on some type, but such a dependency would break dependency management rules.</span></span> <span data-ttu-id="124e8-117">Například závislost z <xref:System.String> k <xref:System.Uri?displayProperty=nameWithType> je pravděpodobně není žádoucí a tak `String.ToUri()` instance metoda vrací `System.Uri` by nesprávný návrhu z hlediska správy závislostí.</span><span class="sxs-lookup"><span data-stu-id="124e8-117">For example, a dependency from <xref:System.String> to <xref:System.Uri?displayProperty=nameWithType> is probably not desirable, and so `String.ToUri()` instance method returning `System.Uri` would be the wrong design from a dependency management perspective.</span></span> <span data-ttu-id="124e8-118">Statické rozšíření metoda `Uri.ToUri(this string str)` vrácení `System.Uri` by mnohem lepší návrh.</span><span class="sxs-lookup"><span data-stu-id="124e8-118">A static extension method `Uri.ToUri(this string str)` returning `System.Uri` would be a much better design.</span></span>  
  
 <span data-ttu-id="124e8-119">**X nepoužívejte** definování rozšiřující metody na <xref:System.Object?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="124e8-119">**X AVOID** defining extension methods on <xref:System.Object?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="124e8-120">VB uživatelé nebudou moci volat tyto metody pro objekt odkazů pomocí syntaxe využívající metody rozšíření.</span><span class="sxs-lookup"><span data-stu-id="124e8-120">VB users will not be able to call such methods on object references using the extension method syntax.</span></span> <span data-ttu-id="124e8-121">Jazyk Visual Basic nepodporuje volání těchto metod, protože deklarace odkaz jako objekt vynutí všechna volání metod na něm být pozdní vázán v jazyce Visual Basic, (skutečného člena s názvem je určena za běhu), zatímco vazby na rozšiřující metody jsou určeny při kompilaci (časná vázaný).</span><span class="sxs-lookup"><span data-stu-id="124e8-121">VB does not support calling such methods because, in VB, declaring a reference as Object forces all method invocations on it to be late bound (actual member called is determined at runtime), while bindings to extension methods are determined at compile-time (early bound).</span></span>  
  
 <span data-ttu-id="124e8-122">Všimněte si, že obecných zásad platí pro další jazyky, kde je přítomen stejné chování vazby, nebo pokud rozšíření metody nejsou podporovány.</span><span class="sxs-lookup"><span data-stu-id="124e8-122">Note that the guideline applies to other languages where the same binding behavior is present, or where extension methods are not supported.</span></span>  
  
 <span data-ttu-id="124e8-123">**X nesmí** umístění rozšiřujících metod v jako typ rozšířené o stejný obor názvů, pokud je pro přidání metody do rozhraní nebo správě závislosti.</span><span class="sxs-lookup"><span data-stu-id="124e8-123">**X DO NOT** put extension methods in the same namespace as the extended type unless it is for adding methods to interfaces or for dependency management.</span></span>  
  
 <span data-ttu-id="124e8-124">**X nepoužívejte** definování dvě nebo více metod rozšíření se stejným podpisem, i v případě, že jsou umístěny v různých oborech názvů.</span><span class="sxs-lookup"><span data-stu-id="124e8-124">**X AVOID** defining two or more extension methods with the same signature, even if they reside in different namespaces.</span></span>  
  
 <span data-ttu-id="124e8-125">**✓ ZVAŽTE** definování rozšiřující metody v jako typ rozšířené o stejný obor názvů, pokud typ je rozhraní, a pokud rozšiřující metody jsou určené pro použití ve většině nebo všem případů.</span><span class="sxs-lookup"><span data-stu-id="124e8-125">**✓ CONSIDER** defining extension methods in the same namespace as the extended type if the type is an interface and if the extension methods are meant to be used in most or all cases.</span></span>  
  
 <span data-ttu-id="124e8-126">**X nesmí** definovat rozšiřující metody, které implementují funkce v obory názvů, které jsou obvykle spojené s jinými funkcemi.</span><span class="sxs-lookup"><span data-stu-id="124e8-126">**X DO NOT** define extension methods implementing a feature in namespaces normally associated with other features.</span></span> <span data-ttu-id="124e8-127">Místo toho je definujte v oboru názvů související s funkcí, ke které patří.</span><span class="sxs-lookup"><span data-stu-id="124e8-127">Instead, define them in the namespace associated with the feature they belong to.</span></span>  
  
 <span data-ttu-id="124e8-128">**X nepoužívejte** obecné názvy oborů názvů vyhrazený pro metody rozšíření (například "rozšíření").</span><span class="sxs-lookup"><span data-stu-id="124e8-128">**X AVOID** generic naming of namespaces dedicated to extension methods (e.g., "Extensions").</span></span> <span data-ttu-id="124e8-129">Použijte popisný název (například "směrování") místo.</span><span class="sxs-lookup"><span data-stu-id="124e8-129">Use a descriptive name (e.g., "Routing") instead.</span></span>
