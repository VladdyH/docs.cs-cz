# <a name="key-takeaways"></a>Stěžejní zjištění

-   Řešení založených na kontejnerech poskytnout důležité výhody úspory nákladů, protože jsou kontejnery, řešení problémů s nasazením kvůli absenci závislosti v produkčním prostředí, proto se výrazně zlepšuje operace DevOps a produkčního prostředí .

-   Docker je stále de facto standardem v odvětví kontejneru, podporovaných nejvýznamnější dodavateli v ekosystémy operačních systémů Linux a Windows, včetně společnosti Microsoft. Docker v budoucnu bude všudypřítomná v libovolné datacentrum do cloudu nebo lokálně.

-   Kontejner Dockeru se standardní jednotkou nasazení pro všechny serverové aplikace nebo služba.

-   Docker orchestrátorů, jako jsou k dispozici v Azure Container Service (Mesosphere Datacenter operačního systému (DC/OS), Docker Swarm, Kubernetes) a Azure Service Fabric jsou základní a nezbytné pro libovolné založených na mikroslužbách nebo vícekontejnerových potřebuje významné složitost a škálovatelnost.

-   Podpora průběžné integrace a průběžného nasazování připojení do produkčních prostředí Dockeru prostředí DevOps začátku do konce poskytuje flexibilitu a nakonec zlepšuje dobu uvedení na trh vašich aplikací.

Služby Azure DevOps výrazně zjednodušuje prostředí DevOps určených k prostředí Dockeru z kanálů průběžné nasazování, včetně jednoduché prostředí Dockeru nebo pokročilejší mikroslužeb a kontejnerů orchestrátorů založené na Azure.
