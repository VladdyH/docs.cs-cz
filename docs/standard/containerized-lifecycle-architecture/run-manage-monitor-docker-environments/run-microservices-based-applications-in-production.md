<span data-ttu-id="5024b-107">Dříve v této e knihy, zavedli jsme *clustery* a *plánovače* jako součást diskusi o architektura softwaru a vývoj.</span><span class="sxs-lookup"><span data-stu-id="5024b-107">Earlier in this e-book, we introduced *clusters* and *schedulers* as part of the discussion on software architecture and development.</span></span> <span data-ttu-id="5024b-108">Příkladem Docker clustery jsou Docker Swarm a Mesosphere Datacenter operačního systému (DC/OS).</span><span class="sxs-lookup"><span data-stu-id="5024b-108">Examples of Docker clusters are Docker Swarm and Mesosphere Datacenter Operating System (DC/OS).</span></span> <span data-ttu-id="5024b-109">Obě tyto můžete spouštět jako součást infrastruktury služby Microsoft Azure Container Service.</span><span class="sxs-lookup"><span data-stu-id="5024b-109">Both of these can run as a part of the infrastructure provided by Microsoft Azure Container Service.</span></span>

<span data-ttu-id="5024b-110">Když aplikace jsou instancemi mezi několika systémy hostitele, stane se atraktivní schopnost spravovat každý hostitelský systém a abstraktní rychle složitost základní platformy.</span><span class="sxs-lookup"><span data-stu-id="5024b-110">When applications are scaled-out across multiple host systems, the ability to manage each host system and abstract away the complexity of the underlying platform becomes attractive.</span></span> <span data-ttu-id="5024b-111">Je přesněji co orchestrators a plánovače poskytují.</span><span class="sxs-lookup"><span data-stu-id="5024b-111">That is precisely what orchestrators and schedulers provide.</span></span> <span data-ttu-id="5024b-112">Podívejme se stručný na je tady:</span><span class="sxs-lookup"><span data-stu-id="5024b-112">Let's take a brief look at them here:</span></span>

-   <span data-ttu-id="5024b-113">**Plánovače *** *"Plánování" odkazuje na schopnost Správce služby soubor do systém hostitele, který určuje jak spustit specifický kontejner.</span><span class="sxs-lookup"><span data-stu-id="5024b-113">**Schedulers*** *"Scheduling" refers to the ability for an administrator to load a service file onto a host system that establishes how to run a specific container.</span></span> <span data-ttu-id="5024b-114">Spuštění kontejnery v clusteru Docker obvykle označují jako plánování.</span><span class="sxs-lookup"><span data-stu-id="5024b-114">Launching containers in a Docker cluster tends to be known as scheduling.</span></span> <span data-ttu-id="5024b-115">I když plánování odkazuje na konkrétní operace načítání definici služby v další obecné smysl, plánovače jsou zodpovědní za zapojování do systému init hostitele ke správě služeb v jakémkoli kapacity potřeby.</span><span class="sxs-lookup"><span data-stu-id="5024b-115">Although scheduling refers to the specific act of loading the service definition, in a more general sense, schedulers are responsible for hooking into a host's init system to manage services in whatever capacity needed.</span></span>

<span data-ttu-id="5024b-116">Scheduler clusteru má více cílů: efektivně pomocí prostředků clusteru, práce s uživatelem zadané umístění omezení, plánování aplikace rychle není ponechat je ve stavu čekající na vyřízení, má určitý stupeň "rovnosti," Probíhá robustní na chyby, a být vždy k dispozici.</span><span class="sxs-lookup"><span data-stu-id="5024b-116">A cluster scheduler has multiple goals: using the cluster's resources efficiently, working with user-supplied placement constraints, scheduling applications rapidly to not leave them in a pending state, having a degree of "fairness," being robust to errors, and always be available.</span></span>

-   <span data-ttu-id="5024b-117">**Orchestrace *** *platformy rozšířit funkce pro správu životního cyklu komplexní, multicontainer úlohy, které jsou nasazené na clusteru hostitelů.</span><span class="sxs-lookup"><span data-stu-id="5024b-117">**Orchestration*** *Platforms extend life-cycle management capabilities to complex, multicontainer workloads deployed on a cluster of hosts.</span></span> <span data-ttu-id="5024b-118">Podle poskytuje abstrakci infrastruktury hostitele, nástroje orchestration uživatelům poskytnout způsob, jak celý cluster považovat za cíl jedno nasazení.</span><span class="sxs-lookup"><span data-stu-id="5024b-118">By abstracting the host infrastructure, orchestration tools give users a way to treat the entire cluster as a single deployment target.</span></span>

<span data-ttu-id="5024b-119">Proces orchestration zahrnuje nástrojů a platformu, která můžete automatizovat všechny aspekty správy aplikací z počáteční umístění nebo nasazení na kontejneru; Přesunutí kontejnery na různých hostitelích v závislosti na jeho hostitel stavu nebo výkonu. Správa verzí kumulativní aktualizace a monitorování funkce, které podporují škálování a převzetí služeb při selhání; stavu a mnoho dalších.</span><span class="sxs-lookup"><span data-stu-id="5024b-119">The process of orchestration involves tooling and a platform that can automate all aspects of application management from initial placement or deployment per container; moving containers to different hosts depending on its host's health or performance; versioning and rolling updates and health monitoring functions that support scaling and failover; and many more.</span></span>

<span data-ttu-id="5024b-120">Orchestrace je široký termín, který odkazuje na kontejner plánování, Správa clusteru a možná zřizování další hostitele.</span><span class="sxs-lookup"><span data-stu-id="5024b-120">Orchestration is a broad term that refers to container scheduling, cluster management, and possibly the provisioning of additional hosts.</span></span>

<span data-ttu-id="5024b-121">Funkce poskytované verzí orchestrators a plánovače jsou velmi složité k vývoji a vytvořit od začátku, a proto obvykle by chcete použití orchestration řešení nabízí Dodavatelé.</span><span class="sxs-lookup"><span data-stu-id="5024b-121">The capabilities provided by orchestrators and schedulers are very complex to develop and create from scratch, and therefore you usually would want to make use of orchestration solutions offered by vendors.</span></span>
