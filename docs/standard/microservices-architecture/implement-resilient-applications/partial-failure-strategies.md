<span data-ttu-id="271ee-107">Naopak, s výjimkou front-end komunikace mezi klientskými aplikacemi a první úroveň mikroslužeb nebo podrobných brány rozhraní API, doporučuje se použít jenom asynchronní <span data-ttu-id="271ee-117">**Použití vzoru jistič**.</span><span class="sxs-lookup"><span data-stu-id="271ee-117">**Use the Circuit Breaker pattern**.</span></span> <span data-ttu-id="271ee-118">V tomto přístupu procesu klienta sleduje počet neúspěšných požadavků.</span><span class="sxs-lookup"><span data-stu-id="271ee-118">In this approach, the client process tracks the number of failed requests.</span></span> <span data-ttu-id="271ee-119">Pokud je míra chyb překračuje limit konfiguraci cest "jistič" tak, aby další pokusy o neúspěšné okamžitě.</span><span class="sxs-lookup"><span data-stu-id="271ee-119">If the error rate exceeds a configured limit, a “circuit breaker” trips so that further attempts fail immediately.</span></span> <span data-ttu-id="271ee-120">(Pokud se nedaří velký počet požadavků, která během psaní navrhuje služba není dostupná a že je zbytečný odesílání požadavků.)
