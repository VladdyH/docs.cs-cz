### <a name="defining-an-event-bus-interface"></a><span data-ttu-id="6b797-168">Definování rozhraní sběrnice událostí</span><span class="sxs-lookup"><span data-stu-id="6b797-168">Defining an event bus interface</span></span>

<span data-ttu-id="6b797-169">Začněme s možné implementace pro účely zkoumání a určitý kód implementace událostí rozhraní sběrnice.</span><span class="sxs-lookup"><span data-stu-id="6b797-169">Let’s start with some implementation code for the event bus interface and possible implementations for exploration purposes.</span></span> <span data-ttu-id="6b797-170">Rozhraní by měl být obecné a jasné, stejně jako následující rozhraní.</span><span class="sxs-lookup"><span data-stu-id="6b797-170">The interface should be generic and straightforward, as in the following interface.</span></span>

```csharp
public interface IEventBus
{
    void Publish(IntegrationEvent @event);

    void Subscribe<T, TH>()
        where T : IntegrationEvent
        where TH : IIntegrationEventHandler<T>;

    void SubscribeDynamic<TH>(string eventName)
        where TH : IDynamicIntegrationEventHandler;

    void UnsubscribeDynamic<TH>(string eventName)
        where TH : IDynamicIntegrationEventHandler;

    void Unsubscribe<T, TH>()
        where TH : IIntegrationEventHandler<T>
        where T : IntegrationEvent;
}
```

<span data-ttu-id="6b797-174">`Subscribe` Metody (může mít několik implementace v závislosti na argumenty) jsou používány mikroslužeb, který chcete přijímat události.</span><span class="sxs-lookup"><span data-stu-id="6b797-174">The `Subscribe` methods (you can have several implementations depending on the arguments) are used by the microservices that want to receive events.</span></span> <span data-ttu-id="6b797-175">Tato metoda má dva argumenty.</span><span class="sxs-lookup"><span data-stu-id="6b797-175">This method has two arguments.</span></span> <span data-ttu-id="6b797-176">První je integrace událostí přihlášení k odběru (`IntegrationEvent`).</span><span class="sxs-lookup"><span data-stu-id="6b797-176">The first is the integration event to subscribe to (`IntegrationEvent`).</span></span> <span data-ttu-id="6b797-177">Druhý argument je integrace událost obslužné rutiny (nebo metoda zpětného volání), s názvem `IIntegrationEventHandler<T>`, budou spuštěny při mikroslužbu příjemce získá této zprávě události integrace.</span><span class="sxs-lookup"><span data-stu-id="6b797-177">The second argument is the integration event handler (or callback method), named `IIntegrationEventHandler<T>`, to be executed when the receiver microservice gets that integration event message.</span></span>
