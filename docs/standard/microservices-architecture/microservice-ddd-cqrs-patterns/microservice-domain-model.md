### <a name="the-aggregate-pattern"></a><span data-ttu-id="2b126-178">Agregační vzor</span><span class="sxs-lookup"><span data-stu-id="2b126-178">The Aggregate pattern</span></span>

<span data-ttu-id="2b126-179">Model domény obsahuje clustery různých datových entit a procesy, které můžete řídit důležité oblasti funkce, například pořadí platbu nebo z inventáře.</span><span class="sxs-lookup"><span data-stu-id="2b126-179">A domain model contains clusters of different data entities and processes that can control a significant area of functionality, such as order fulfilment or inventory.</span></span> <span data-ttu-id="2b126-180">Podrobnějšího jednotka DDD je agregaci, která popisuje cluster nebo skupinu entit a chování, které mohou být považovány za získá na ucelenosti jednotku.</span><span class="sxs-lookup"><span data-stu-id="2b126-180">A more fine-grained DDD unit is the aggregate, which describes a cluster or group of entities and behaviors that can be treated as a cohesive unit.</span></span>

<span data-ttu-id="2b126-181">Obvykle definujete agregaci podle transakcí, které potřebujete.</span><span class="sxs-lookup"><span data-stu-id="2b126-181">You usually define an aggregate based on the transactions that you need.</span></span> <span data-ttu-id="2b126-182">Classic příkladem je pořadí, které obsahuje také pořadí položek seznamu.</span><span class="sxs-lookup"><span data-stu-id="2b126-182">A classic example is an order that also contains a list of order items.</span></span> <span data-ttu-id="2b126-183">Položku pořadí bude obvykle entity.</span><span class="sxs-lookup"><span data-stu-id="2b126-183">An order item will usually be an entity.</span></span> <span data-ttu-id="2b126-184">Ale bude podřízené entity v rámci pořadí agregace, který bude také obsahovat entity pořadí jako jeho kořenový entitu, obvykle nazývá agregační root.</span><span class="sxs-lookup"><span data-stu-id="2b126-184">But it will be a child entity within the order aggregate, which will also contain the order entity as its root entity, typically called an aggregate root.</span></span>

<span data-ttu-id="2b126-185">Identifikace agregace může být složité.</span><span class="sxs-lookup"><span data-stu-id="2b126-185">Identifying aggregates can be hard.</span></span> <span data-ttu-id="2b126-186">Agregace je pro skupinu objektů, které musí být konzistentní společně, ale nemůže právě vyberte skupinu objektů a označovat je agregace.</span><span class="sxs-lookup"><span data-stu-id="2b126-186">An aggregate is a group of objects that must be consistent together, but you cannot just pick a group of objects and label them an aggregate.</span></span> <span data-ttu-id="2b126-187">Musí začínat koncept domény a vezměte v úvahu entitami, které se používají v nejběžnější transakce související pro daný koncept.</span><span class="sxs-lookup"><span data-stu-id="2b126-187">You must start with a domain concept and think about the entities that are used in the most common transactions related to that concept.</span></span> <span data-ttu-id="2b126-188">Tyto entity, které musí být stavu transakční konzistence se, co tvoří agregace.</span><span class="sxs-lookup"><span data-stu-id="2b126-188">Those entities that need to be transactionally consistent are what forms an aggregate.</span></span> <span data-ttu-id="2b126-189">Zamyšlení operací transakce je pravděpodobně nejlepší způsob, jak identifikovat agregace.</span><span class="sxs-lookup"><span data-stu-id="2b126-189">Thinking about transaction operations is probably the best way to identify aggregates.</span></span>

### <a name="the-aggregate-root-or-root-entity-pattern"></a><span data-ttu-id="2b126-190">Vzor agregační kořenovou nebo kořenové Entity</span><span class="sxs-lookup"><span data-stu-id="2b126-190">The Aggregate Root or Root Entity pattern</span></span>

<span data-ttu-id="2b126-191">Agregace se skládá z nejméně jedna entita: agregační kořenový adresář, označovaný taky jako kořenové entity nebo primární entity.</span><span class="sxs-lookup"><span data-stu-id="2b126-191">An aggregate is composed of at least one entity: the aggregate root, also called root entity or primary entity.</span></span> <span data-ttu-id="2b126-192">Kromě toho může mít více podřízených entit a hodnota objekty s všechny entity a objekty, které pracují společně k implementaci požadované chování a transakce.</span><span class="sxs-lookup"><span data-stu-id="2b126-192">Additionally, it can have multiple child entities and value objects, with all entities and objects working together to implement required behavior and transactions.</span></span>

<span data-ttu-id="2b126-193">Účelem agregační kořenové je zajištění konzistence agregace; je nutné pouze vstupní bod pro aktualizace agregace prostřednictvím metody nebo operace v souhrnném kořenová třída.</span><span class="sxs-lookup"><span data-stu-id="2b126-193">The purpose of an aggregate root is to ensure the consistency of the aggregate; it should be the only entry point for updates to the aggregate through methods or operations in the aggregate root class.</span></span> <span data-ttu-id="2b126-194">Entity v rámci agregace pouze prostřednictvím kořenu agregační měli provádět změny.</span><span class="sxs-lookup"><span data-stu-id="2b126-194">You should make changes to entities within the aggregate only via the aggregate root.</span></span> <span data-ttu-id="2b126-195">Je na agregaci konzistence ochrany, vezme v úvahu výstupních podmínek a konzistence pravidla, která může být nutné v souladu se ve vašem agregace.</span><span class="sxs-lookup"><span data-stu-id="2b126-195">It is the aggregate’s consistency guardian, taking into account all the invariants and consistency rules you might need to comply with in your aggregate.</span></span> <span data-ttu-id="2b126-196">Pokud změníte podřízený objekt entity nebo hodnota nezávisle, kořenu agregační nelze zajistěte, aby byl agregace v platném stavu.</span><span class="sxs-lookup"><span data-stu-id="2b126-196">If you change a child entity or value object independently, the aggregate root cannot ensure that the aggregate is in a valid state.</span></span> <span data-ttu-id="2b126-197">Je jako tabulku s přijít větev.</span><span class="sxs-lookup"><span data-stu-id="2b126-197">It would be like a table with a loose leg.</span></span> <span data-ttu-id="2b126-198">Údržba konzistence je hlavním účelem agregační kořenu.</span><span class="sxs-lookup"><span data-stu-id="2b126-198">Maintaining consistency is the main purpose of the aggregate root.</span></span>
