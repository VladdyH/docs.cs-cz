## <a name="general-principles"></a><span data-ttu-id="666e8-107">Obecné zásady</span><span class="sxs-lookup"><span data-stu-id="666e8-107">General Principles</span></span>

## <a name="guidelines"></a><span data-ttu-id="666e8-112">Pokyny</span><span class="sxs-lookup"><span data-stu-id="666e8-112">Guidelines</span></span>  
 <span data-ttu-id="666e8-113">Následující seznam popisuje pokyny pro při měli byste implementovat asynchronní vzor na základě událostí:</span><span class="sxs-lookup"><span data-stu-id="666e8-113">The following list describes the guidelines for when you should implement Event-based Asynchronous Pattern:</span></span>

-   <span data-ttu-id="666e8-115">Nevystavujte <xref:System.IAsyncResult> vzor, když vaše třída se používá hlavně v aplikaci klienta, například Windows Forms.</span><span class="sxs-lookup"><span data-stu-id="666e8-115">Do not expose the <xref:System.IAsyncResult> pattern when your class is primarily used in a client application, for example Windows Forms.</span></span>  
  
-   <span data-ttu-id="666e8-116">Pouze vystavit <xref:System.IAsyncResult> vzor, když je to nutné pro splnění požadavků.</span><span class="sxs-lookup"><span data-stu-id="666e8-116">Only expose the <xref:System.IAsyncResult> pattern when it is necessary for meeting your requirements.</span></span> <span data-ttu-id="666e8-117">Pro kompatibilitu s existujícího rozhraní API může třeba vyžadovat, abyste vystavit <xref:System.IAsyncResult> vzor.</span><span class="sxs-lookup"><span data-stu-id="666e8-117">For example, compatibility with an existing API may require you to expose the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="666e8-118">Nevystavujte <xref:System.IAsyncResult> vzor bez také vystavení vzor na základě událostí.</span><span class="sxs-lookup"><span data-stu-id="666e8-118">Do not expose the <xref:System.IAsyncResult> pattern without also exposing the event-based pattern.</span></span>  
  
-   <span data-ttu-id="666e8-119">Pokud musí vystavit <xref:System.IAsyncResult> vzor, učinit jako upřesňující možnost.</span><span class="sxs-lookup"><span data-stu-id="666e8-119">If you must expose the <xref:System.IAsyncResult> pattern, do so as an advanced option.</span></span> <span data-ttu-id="666e8-120">Například pokud vygenerování objekt proxy vygenerovat vzor na základě událostí ve výchozím nastavení s možností generovat <xref:System.IAsyncResult> vzor.</span><span class="sxs-lookup"><span data-stu-id="666e8-120">For example, if you generate a proxy object, generate the event-based pattern by default, with an option to generate the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="666e8-121">Sestavení vaší implementace vzoru založeného na události váš <xref:System.IAsyncResult> vzor implementace.</span><span class="sxs-lookup"><span data-stu-id="666e8-121">Build your event-based pattern implementation on your <xref:System.IAsyncResult> pattern implementation.</span></span>  
  
-   <span data-ttu-id="666e8-122">Vyhněte se vystavení obou vzor na základě událostí a <xref:System.IAsyncResult> vzor pro stejnou třídu.</span><span class="sxs-lookup"><span data-stu-id="666e8-122">Avoid exposing both the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class.</span></span> <span data-ttu-id="666e8-123">Vystavení vzor na základě událostí na "vyšší úroveň" třídy a <xref:System.IAsyncResult> vzor na "nižší úroveň" třídy.</span><span class="sxs-lookup"><span data-stu-id="666e8-123">Expose the event-based pattern on "higher level" classes and the <xref:System.IAsyncResult> pattern on "lower level" classes.</span></span> <span data-ttu-id="666e8-124">Například porovnat vzor na základě událostí na <xref:System.Net.WebClient> součásti s <xref:System.IAsyncResult> vzor na <xref:System.Web.HttpRequest> třídy.</span><span class="sxs-lookup"><span data-stu-id="666e8-124">For example, compare the event-based pattern on the <xref:System.Net.WebClient> component with the <xref:System.IAsyncResult> pattern on the <xref:System.Web.HttpRequest> class.</span></span>  
  
    -   <span data-ttu-id="666e8-125">Vystavení vzor na základě událostí a <xref:System.IAsyncResult> vzor pro stejnou třídu při kompatibility vyžaduje.</span><span class="sxs-lookup"><span data-stu-id="666e8-125">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class when compatibility requires it.</span></span> <span data-ttu-id="666e8-126">Například, pokud již máte vydané rozhraní API, které používá <xref:System.IAsyncResult> vzor, museli byste zachovat <xref:System.IAsyncResult> vzor z důvodu zpětné kompatibility.</span><span class="sxs-lookup"><span data-stu-id="666e8-126">For example, if you have already released an API that uses the <xref:System.IAsyncResult> pattern, you would need to retain the <xref:System.IAsyncResult> pattern for backward compatibility.</span></span>  
  
    -   <span data-ttu-id="666e8-127">Vystavení vzor na základě událostí a <xref:System.IAsyncResult> vzor u stejné třídy, pokud výsledný objekt modelu složitost převáží výhodou oddělení implementace.</span><span class="sxs-lookup"><span data-stu-id="666e8-127">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class if the resulting object model complexity outweighs the benefit of separating the implementations.</span></span> <span data-ttu-id="666e8-128">Je lepší vystavit obou vzory u jedné třídy, než se vyhnout vystavení vzor na základě událostí.</span><span class="sxs-lookup"><span data-stu-id="666e8-128">It is better to expose both patterns on a single class than to avoid exposing the event-based pattern.</span></span>  
  
    -   <span data-ttu-id="666e8-129">Pokud musí vystavit obou vzor na základě událostí a <xref:System.IAsyncResult> vzor na jednu třídu, použijte <xref:System.ComponentModel.EditorBrowsableAttribute> nastavena na <xref:System.ComponentModel.EditorBrowsableState.Advanced> označit <xref:System.IAsyncResult> vzor implementace jako upřesňující funkce.</span><span class="sxs-lookup"><span data-stu-id="666e8-129">If you must expose both the event-based pattern and <xref:System.IAsyncResult> pattern on a single class, use <xref:System.ComponentModel.EditorBrowsableAttribute> set to <xref:System.ComponentModel.EditorBrowsableState.Advanced> to mark the <xref:System.IAsyncResult> pattern implementation as an advanced feature.</span></span> <span data-ttu-id="666e8-130">To znamená do návrhu prostředí, jako například [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)] technologii IntelliSense, není pro zobrazení <xref:System.IAsyncResult> vlastnosti a metody.</span><span class="sxs-lookup"><span data-stu-id="666e8-130">This indicates to design environments, such as [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)] IntelliSense, not to display the <xref:System.IAsyncResult> properties and methods.</span></span> <span data-ttu-id="666e8-131">Tyto vlastnosti a metody jsou stále plně použitelná, ale má vývojář pracující pomocí IntelliSense většího přehledu rozhraní API.</span><span class="sxs-lookup"><span data-stu-id="666e8-131">These properties and methods are still fully usable, but the developer working through IntelliSense has a clearer view of the API.</span></span>  
  
## <a name="criteria-for-exposing-the-iasyncresult-pattern-in-addition-to-the-event-based-pattern"></a><span data-ttu-id="666e8-132">Kritéria pro vystavení vzoru IAsyncResult kromě vzor založený na událostech</span><span class="sxs-lookup"><span data-stu-id="666e8-132">Criteria for Exposing the IAsyncResult Pattern in Addition to the Event-based Pattern</span></span>  
 <span data-ttu-id="666e8-133">Během asynchronního vzoru založeného na událostech má mnoho výhod v rámci výše uvedené scénáře, nemá některé nevýhody, které byste měli vědět, pokud je vaše nejdůležitější požadavek na výkon.</span><span class="sxs-lookup"><span data-stu-id="666e8-133">While the Event-based Asynchronous Pattern has many benefits under the previously mentioned scenarios, it does have some drawbacks, which you should be aware of if performance is your most important requirement.</span></span>  
  
 <span data-ttu-id="666e8-134">Existují tři scénáře, které vzor na základě událostí neřeší společně s <xref:System.IAsyncResult> vzoru:</span><span class="sxs-lookup"><span data-stu-id="666e8-134">There are three scenarios that the event-based pattern does not address as well as the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="666e8-135">Blokování čekání na jednom<xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="666e8-135">Blocking wait on one <xref:System.IAsyncResult></span></span>  
  
-   <span data-ttu-id="666e8-136">Blokování čekání na mnoho <xref:System.IAsyncResult> objekty</span><span class="sxs-lookup"><span data-stu-id="666e8-136">Blocking wait on many <xref:System.IAsyncResult> objects</span></span>  
  
-   <span data-ttu-id="666e8-137">Dotazování pro dokončení na<xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="666e8-137">Polling for completion on the <xref:System.IAsyncResult></span></span>  
  
 <span data-ttu-id="666e8-138">Tyto scénáře můžete vyřešit tak, že pomocí vzoru založeného na událostech, ale to se těžkopádnější než při použití <xref:System.IAsyncResult> vzor.</span><span class="sxs-lookup"><span data-stu-id="666e8-138">You can address these scenarios by using the event-based pattern, but doing so is more cumbersome than using the <xref:System.IAsyncResult> pattern.</span></span>  
  
 <span data-ttu-id="666e8-139">Vývojáři často používají <xref:System.IAsyncResult> vzor pro služby, které obvykle mají požadavky na velmi vysoký výkon.</span><span class="sxs-lookup"><span data-stu-id="666e8-139">Developers often use the <xref:System.IAsyncResult> pattern for services that typically have very high performance requirements.</span></span> <span data-ttu-id="666e8-140">Například dotazování pro dokončení scénář je technika vysoce výkonné serveru.</span><span class="sxs-lookup"><span data-stu-id="666e8-140">For example, the polling for completion scenario is a high-performance server technique.</span></span>  
  
 <span data-ttu-id="666e8-141">Kromě toho je míň efektivní než vzor na základě událostí <xref:System.IAsyncResult> vzor, protože se tím vytvoří další objekty, zejména <xref:System.EventArgs>, a protože synchronizuje napříč vlákny.</span><span class="sxs-lookup"><span data-stu-id="666e8-141">Additionally, the event-based pattern is less efficient than the <xref:System.IAsyncResult> pattern because it creates more objects, especially <xref:System.EventArgs>, and because it synchronizes across threads.</span></span>  
  
 <span data-ttu-id="666e8-142">Následující seznam uvádí několik doporučení, které řídí, pokud se rozhodnete použít <xref:System.IAsyncResult> vzoru:</span><span class="sxs-lookup"><span data-stu-id="666e8-142">The following list shows some recommendations to follow if you decide to use the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="666e8-143">Pouze vystavit <xref:System.IAsyncResult> vzor, pokud požadujete konkrétně podpora <xref:System.Threading.WaitHandle> nebo <xref:System.IAsyncResult> objekty.</span><span class="sxs-lookup"><span data-stu-id="666e8-143">Only expose the <xref:System.IAsyncResult> pattern when you specifically require support for <xref:System.Threading.WaitHandle> or <xref:System.IAsyncResult> objects.</span></span>  
  
-   <span data-ttu-id="666e8-144">Pouze vystavit <xref:System.IAsyncResult> vzor, pokud máte existující rozhraní API, které používá <xref:System.IAsyncResult> vzor.</span><span class="sxs-lookup"><span data-stu-id="666e8-144">Only expose the <xref:System.IAsyncResult> pattern when you have an existing API that uses the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="666e8-145">Pokud máte existujícího rozhraní API na základě <xref:System.IAsyncResult> vzor, zvažte také vystavení vzor na základě událostí v příštím vydání.</span><span class="sxs-lookup"><span data-stu-id="666e8-145">If you have an existing API based on the <xref:System.IAsyncResult> pattern, consider also exposing the event-based pattern in your next release.</span></span>  
  
-   <span data-ttu-id="666e8-146">Pouze vystavit <xref:System.IAsyncResult> vzor, pokud máte požadavky vysoký výkon, které jste ověřili nelze splnit vzor na základě událostí, ale můžete splnit <xref:System.IAsyncResult> vzor.</span><span class="sxs-lookup"><span data-stu-id="666e8-146">Only expose <xref:System.IAsyncResult> pattern if you have high performance requirements which you have verified cannot be met by the event-based pattern but can be met by the <xref:System.IAsyncResult> pattern.</span></span>
