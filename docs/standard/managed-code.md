## <a name="intermediate-language--execution"></a><span data-ttu-id="ba4b3-119">Zprostředkující jazyk a provádění</span><span class="sxs-lookup"><span data-stu-id="ba4b3-119">Intermediate Language & execution</span></span>

<span data-ttu-id="ba4b3-129">Samozřejmě modulu CLR umožňuje předávání hranice mezi spravovanými a nespravovanými world, a je mnoha kód, který nemá tedy i při [základní knihovny tříd](framework-libraries.md).</span><span class="sxs-lookup"><span data-stu-id="ba4b3-129">Of course, the CLR allows passing the boundaries between managed and unmanaged world, and there is a lot of code that does that, even in the [Base Class Libraries](framework-libraries.md).</span></span> <span data-ttu-id="ba4b3-130">To se označuje jako **interoperabilita** nebo právě **zprostředkovatel komunikace s objekty** pro krátké.</span><span class="sxs-lookup"><span data-stu-id="ba4b3-130">This is called **interoperability** or just **interop** for short.</span></span> <span data-ttu-id="ba4b3-131">Tato ustanovení by umožnilo, například zabalit do nespravovaných knihovny a volání do ní.</span><span class="sxs-lookup"><span data-stu-id="ba4b3-131">These provisions would allow you to, for example, wrap up an unmanaged library and call into it.</span></span> <span data-ttu-id="ba4b3-132">Je ale důležité si uvědomit, že jakmile to uděláte, když kód předá hranice modul runtime, skutečných spuštění je znovu ruční nespravovaného kódu a proto klesne pod stejná omezení.</span><span class="sxs-lookup"><span data-stu-id="ba4b3-132">However, it is important to note that once you do this, when the code passes the boundaries of the runtime, the actual management of the execution is again in the hand of unmanaged code, and thus falls under the same restrictions.</span></span>
