### <a name="product-versioning-changes-in-the-net-framework-46-and-later-versions"></a>Změny verze produktu v rozhraní .NET Framework 4.6 a novějších verzích

|   |   |
|---|---|
|Podrobnosti|Správa verzí produktu se změnila z předchozích verzí rozhraní .NET Framework a zejména z rozhraní .NET Framework 4, 4.5, 4.5.1, a 4.5.2.The následují podrobné změny:<ul><li>Hodnota <code>Version</code> položku v <code>HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full</code> klíče se změnila na <code>4.6.xxxxx</code> pro rozhraní .NET Framework 4.6 a jeho verze bodu a <code>4.7.xxxxx</code> 4.7 rozhraní .NET Framework a 4.7.1. V rozhraní .NET Framework 4.5 a 4.5.1, 4.5.2, měl formát <code>4.5.xxxxx</code>.</li><li>Správa verzí souborů a produktu pro soubory rozhraní .NET Framework se změnil ze starší verze schéma 4.0.30319.x 4.6.X.0 pro rozhraní .NET Framework 4.6 a jeho verze bodu a 4.7.X.0 pro 4.7 rozhraní .NET Framework a 4.7.1. Zobrazí se tyto nové hodnoty a při zobrazení vlastností souboru po pravým tlačítkem myši na soubor.</li><li><xref:System.Reflection.AssemblyFileVersionAttribute> a <xref:System.Reflection.AssemblyInformationalVersionAttribute> atributy pro spravovaná sestavení mít hodnoty verze ve formě 4.6.X.0 pro rozhraní .NET Framework 4.6 a jeho verze bodu a 4.7.X.0 4.7 rozhraní .NET Framework a 4.7.1.</li><li>V rozhraní .NET Framework 4.6, 4.6.1, 4.6.2, 4.7 a 4.7.1 <xref:System.Environment.Version?displayProperty=nameWithType> vlastnost vrací řetězec pevné verze <code>4.0.30319.42000</code>. V rozhraní .NET Framework 4, 4.5, 4.5.1 a 4.5.2, vrátí verze řetězce ve formátu <code>4.0.30319.xxxxx</code> (například &quot;4.0.30319.18010&quot;). Všimněte si, že nedoporučujeme trvá žádné nové závislosti na vlastnosti Environment.Version kódu aplikace.</li></ul>Další informace najdete v tématu [postupy: určení, které nainstalované rozhraní .NET Framework verze jsou](~/docs/framework/migration-guide/how-to-determine-which-versions-are-installed.md).|
|Návrh|Obecně platí aplikace by měla závisí na doporučené postupy pro zjišťování prvků jako verzi modulu runtime rozhraní .NET Framework a instalační adresář:<ul><li>Pokud chcete zjistit verzi modulu runtime rozhraní .NET Framework, přečtěte si téma [postupy: určení rozhraní .NET Framework verze jsou nainstalované](~/docs/framework/migration-guide/how-to-determine-which-versions-are-installed.md).</li><li>Pokud chcete zjistit instalační cesta pro rozhraní .NET Framework, použijte hodnotu <code>InstallPath</code> položku v <code>HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full</code> klíč.</li></ul> <blockquote> [!IMPORTANT] Název podklíče je <code>NET Framework Setup</code>, nikoli <code>.NET Framework Setup</code>.</blockquote> <ul><li>Chcete-li zjistit, cesta k adresáři modulu CLR rozhraní .NET Framework, zavolejte <xref:System.Runtime.InteropServices.RuntimeEnvironment.GetRuntimeDirectory?displayProperty=nameWithType> metoda.</li><li>Chcete-li získat verze CLR, zavolejte <xref:System.Runtime.InteropServices.RuntimeEnvironment.GetSystemVersion?displayProperty=nameWithType> metoda. Pro rozhraní .NET Framework 4 a jeho bod uvolní (rozhraní .NET Framework 4.5, 4.5.1, 4.5.2 a rozhraní .NET Framework 4.6, 4.6.1, 4.6.2, 4.7 a 4.7.1), vrátí v4.0.30319 řetězec.</li></ul>|
