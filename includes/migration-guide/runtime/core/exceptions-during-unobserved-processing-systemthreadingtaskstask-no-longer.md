### <a name="exceptions-during-unobserved-processing-in-systemthreadingtaskstask-no-longer-propagate-on-finalizer-thread"></a>Výjimky při zmeškané zpracování v System.Threading.Tasks.Task už rozšíří na finalizační metodu přístup z více vláken

|   |   |
|---|---|
|Podrobnosti|Protože <xref:System.Threading.Tasks.Task?displayProperty=name> třída reprezentuje asynchronní operaci, zachytávalo všechny-závažné výjimky, ke kterým došlo během asynchronního zpracování. V rozhraní .NET Framework 4.5 Pokud není dodržen výjimku a kódu nikdy čeká na úlohy, výjimka již nebude možné rozšířit na vlákno finalizační metodu a havárií proces během uvolňování paměti. Tato změna zlepšuje spolehlivost aplikací, které pomocí třídy úlohy provádět zmeškané asynchronní zpracování.|
|Návrh|Pokud aplikace závisí na zmeškané asynchronní výjimky šíření finalizační metodu vlákno, může být obnovena předchozí chování tím, že poskytuje příslušnou obslužnou rutinu pro <xref:System.Threading.Tasks.TaskScheduler.UnobservedTaskException> události, nebo nastavením [element konfigurace runtime ](~/docs/framework/configure-apps/file-schema/runtime/throwunobservedtaskexceptions-element.md).|
|Rozsah|Edge|
|Version|4.5|
|Typ|modul runtime|
