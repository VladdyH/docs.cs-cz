### <a name="accessibility-improvements-in-wpf"></a>Vylepšení přístupnosti v grafickém subsystému WPF

|   |   |
|---|---|
|Podrobnosti|<strong>Vylepšení vysoký kontrast</strong></br><ul><li>Fokus pro <xref:System.Windows.Controls.Expander> ovládací prvek je nyní viditelné. V předchozích verzích rozhraní .NET Framework není.</li><li>Text v <xref:System.Windows.Controls.CheckBox> a <xref:System.Windows.Controls.RadioButton> ovládacích prvků, když je tato možnost vybrána, je nyní snazší zjistit než v předchozích verzích rozhraní .NET Framework.</li><li>Ohraničení zakázáno <xref:System.Windows.Controls.ComboBox> je nyní stejné barvy jako zakázaný text. V předchozích verzích rozhraní .NET Framework není.</li><li>Tlačítka Zakázané a zaměřují se teď použít něm barevný motiv správné. V předchozích verzích rozhraní .NET Framework je nepodporovaly.</li><li>Rozevírací tlačítka je nyní viditelné při <xref:System.Windows.Controls.ComboBox> styl ovládacího prvku je nastaven na <xref:System.Windows.Controls.ToolBar.ComboBoxStyleKey?displayProperty=nameWithType>, v předchozích verzích rozhraní .NET Framework není.</li><li>Šipku řazení indikátoru v <xref:System.Windows.Controls.DataGrid> řízení teď používá barev motivů. V předchozích verzích rozhraní .NET Framework je nepodporovaly.</li><li>Výchozí styl hypertextový odkaz teď změny barvu správné motiv na myši. V předchozích verzích rozhraní .NET Framework je nepodporovaly.</li><li>Fokus klávesnice na přepínačů je nyní viditelné. V předchozích verzích rozhraní .NET Framework není.</li><li><xref:System.Windows.Controls.DataGrid> Ovládacího prvku checkbox sloupec teď používá očekávané barvy pro zpětnou vazbu fokus klávesnice. V předchozích verzích rozhraní .NET Framework je nepodporovaly.</li><li>vizuály fokus klávesnice, se nyní zobrazují <xref:System.Windows.Controls.ComboBox> a <xref:System.Windows.Controls.ListBox> ovládací prvky. V předchozích verzích rozhraní .NET Framework není.</p></li></ul><strong>Vylepšení interakce čtečky obrazovky</strong></br><ul><li><xref:System.Windows.Controls.Expander> ovládací prvky jsou nyní správně oznamují jako skupiny (rozbalit nebo sbalit) čtečky obrazovky.</li><li><xref:System.Windows.Controls.DataGridCell> ovládací prvky jsou nyní správně oznámila jako buňky v mřížce dat (lokalizované) čtečky obrazovky.</li><li>Čtečky obrazovky bude nyní oznamujeme název upravitelné <xref:System.Windows.Controls.ComboBox>.</li><li><xref:System.Windows.Controls.PasswordBox> ovládací prvky jsou již oznámeno jako &quot;žádná položka v zobrazení&quot; pomocí čtečky obrazovky.</p></li></ul><strong>Podpora LiveRegion</strong></br>Čtečky obrazovky například Předčítání umožňují uživatelům vědět obsah uživatelského rozhraní aplikace, obvykle prostřednictvím popisu něco o uživatelské rozhraní, které se aktuálně zaměřuje, protože to je pravděpodobně element většina zájmu uživateli. Ale pokud prvku uživatelského rozhraní se změní někde na obrazovce a nemá fokus, uživatel nemusí informováni a neproběhly důležité informace. LiveRegions jsou určené k vyřešení tohoto problému. Vývojář může použít k informování čtečky obrazovky nebo jakékoliv [automatizace uživatelského rozhraní](~/docs/framework/ui-automation/ui-automation-overview.md) klienta, který byl proveden důležité změně prvku uživatelského rozhraní. Čtečky obrazovky můžete pak rozhodnout, jak a kdy k informování uživatelů této změny. Vlastnost LiveSetting také umožňuje čtečky obrazovky vědět, jak je důležité informovat uživatele změny provedené v uživatelském rozhraní.|
|Návrh|<strong>Postup aktivování nebo zrušení těchto změn</strong></br>Aby aplikace, abyste mohli využívat výhod tyto změny, musí běžet na rozhraní .NET Framework 4.7.1 nebo novější. Aplikace můžete využít tyto změny v některém z následujících způsobů:<ul><li>Cílové rozhraní .NET Framework 4.7.1. Toto je doporučený postup. Tyto změny usnadnění jsou povolené ve výchozím nastavení v aplikacích WPF, cílených na rozhraní .NET Framework 4.7.1 nebo novější.</li><li>Ho výslovný nesouhlas chování starší verze usnadnění přidáním následující [AppContext přepínač](~/docs/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element.md) v <code>&lt;runtime&gt;</code> části konfiguračním souboru aplikace a jeho nastavení na hodnotu <code>false</code>, jak ukazuje následující příklad.</li></ul><pre><code class="lang-xml">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;&#13;&#10;&lt;configuration&gt;&#13;&#10;&lt;startup&gt;&#13;&#10;&lt;supportedRuntime version=&quot;v4.0&quot; sku=&quot;.NETFramework,Version=v4.7&quot;/&gt;&#13;&#10;&lt;/startup&gt;&#13;&#10;&lt;runtime&gt;&#13;&#10;&lt;!-- AppContextSwitchOverrides value attribute is in the form of &#39;key1=true/false;key2=true/false  --&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;&lt;/configuration&gt;&#13;&#10;</code></pre>Aplikace, které cílí na rozhraní .NET Framework 4.7.1 nebo novější a chcete zachovat starší verze usnadnění chování můžete vyjádřit výslovný souhlas s použitím funkce usnadnění starší verze explicitně nastavením tento přepínač AppContext na <code>true</code>.</br>Přehled automatizace uživatelského rozhraní, najdete [Přehled automatizace uživatelského rozhraní](~/docs/framework/ui-automation/ui-automation-overview.md).|
