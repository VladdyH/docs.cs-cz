### <a name="tls-1x-by-default-passes-the-schsendauxrecord-flag-to-the-underlying-schannel-api"></a>Protokol TLS 1.x ve výchozím nastavení předá SCH_SEND_AUX_RECORD příznak základního rozhraní API SCHANNEL

|   |   |
|---|---|
|Podrobnosti|Při použití protokolu TLS 1.x, rozhraní .NET Framework spoléhá na rozhraní API základní SCHANNEL systému Windows. Od verze rozhraní .NET Framework 4.6 [SCH_SEND_AUX_RECORD](https://msdn.microsoft.com/library/windows/desktop/aa379810.aspx) příznak předaný ve výchozím nastavení SCHANNEL. To způsobí, že SCHANNEL rozdělit do dvou samostatných záznamů, první jako jediný bajt a druhý jako šifrovat data <em>n</em>-1 bajtů. Ve výjimečných případech dělí toto komunikace mezi klienty a existujících serverů, které se předpokládá, že se data nachází v jeden záznam.|
|Návrh|Pokud je tato změna dělí komunikace s existujícím serverem, můžete zakázat odesílání [SCH_SEND_AUX_RECORD](https://msdn.microsoft.com/library/windows/desktop/aa379810.aspx) příznak a obnovit předchozí chování není rozdělení dat na samostatné záznamy přidáním následující přepínač tak, aby [ \<AppContextSwitchOverrides >](~/docs/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element.md) v [ ` ](~/docs/framework/configure-apps/file-schema/runtime/runtime-element.md) konfiguračního souboru aplikace:<pre><code class="language-xml">&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides&#13;&#10;value=&quot;Switch.System.Net.DontEnableSchSendAuxRecord=true&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;</code></pre> <blockquote> [!IMPORTANT] Toto nastavení je dostupné pouze z důvodů zpětné kompatibility. V opačném případě se nedoporučuje používat.</blockquote> |
